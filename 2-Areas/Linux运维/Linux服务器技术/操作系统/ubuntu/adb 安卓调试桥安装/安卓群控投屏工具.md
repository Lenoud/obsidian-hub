# Appium config
```bash
要连接指定的 Android 设备，你需要提供更多的设备信息，以便 Appium 能够识别并与之通信。下面是一个连接指定设备的示例请求，包含了设备的一些关键信息：

{
  "platformName": "Android",
  "deviceName": "deviceName",
  "platformVersion": "platformVersion",
  "udid": "deviceUDID",
  "appPackage": "com.example",
  "appActivity": ".MainActivity",
  "app":"c:/app.apk"
}
deviceName：设备的名称，可以在设备的系统设置中找到。
platformVersion：设备的 Android 版本，也可以在系统设置中找到。
udid：设备的唯一标识符（UUID），可以通过运行 adb devices 命令来获取。
appPackage：你要测试的应用程序的包名。
app:要下载的apk路径

#adb shell dumpsys package <package_name> | Select-String -Pattern "android.intent.action.MAIN"
appActivity：应用程序的主活动（Main Activity）。
#https://blog.csdn.net/atnanyang/article/details/86608763
/wd/hub/
```

```bash
{
  "platformName": "Android",
  "platformVersion": "6.0",
  "deviceName": "Android Emulator",
  "automationName": "Appium",
  "app": "/Users/cosetteqi/Downloads/llsstaging60.apk",
  "noReset": true,
  "autoGrantPermissions": true,
  "udid": "192.168.56.101:5555"
}

platformName: 指定了测试的平台，这里是Android。

platformVersion: 指定了Android平台的版本，这里是6.0。

deviceName: 指定了设备的名称，可以是任意值，用于识别连接到计算机上的设备，这里是"Android Emulator"。

automationName: 指定了自动化测试工具的名称，这里是Appium，表示使用Appium进行测试。

app: 指定了被测试应用的路径，这里是"/Users/cosetteqi/Downloads/llsstaging60.apk"，表示测试的是一个名为"llsstaging60"的APK文件。

noReset: 指定了测试时是否重置应用的状态，true表示不重置，false表示重置。这里是true，表示不重置应用状态。

autoGrantPermissions: 指定了是否自动授予应用权限，true表示自动授予权限，false表示不自动授予权限。这里是true，表示自动授予权限。

udid: 指定了设备的唯一标识符（UDID），用于连接到指定的设备进行测试，这里是"192.168.56.101:5555"。
```

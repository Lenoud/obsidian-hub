# ext2/3/4 文件系统数据恢复

Linux ext 系列文件系统(ext2 / ext3 / ext4)误删文件后的恢复方法与工具。**重要原则**:发现误删后立刻卸载分区或只读挂载,避免被新数据覆盖。

## 恢复工具对比

| 工具 | 支持文件系统 | 特点 | 适用场景 |
| --- | --- | --- | --- |
| **extundelete** | ext3 / ext4 | 基于 journal 日志恢复,无需付费 | 误删单个文件 / 目录,刚删除不久 |
| **testdisk** | 几乎所有 FS | 支持分区表恢复 + 文件恢复 | 整个分区丢失、分区表损坏 |
| **photorec**(testdisk 附带) | 所有 FS | 按文件头签名恢复,不依赖目录 | ext 格式严重损坏,目录结构丢失 |
| **R-Linux** | ext2 / ext3 / ext4 | 商业软件(免费版有功能限制),GUI | Windows 上恢复 Linux 分区 |
| **debugfs** | ext2 / ext3 / ext4 | 内核自带,命令行交互 | 手动探索 inode、低级恢复 |

## extundelete(推荐首选)

### 安装

```bash
# Debian / Ubuntu
apt install extundelete

# CentOS / RHEL
yum install extundelete
# 或从源码编译(依赖 e2fsprogs-devel)
```

### 基本用法

```bash
# 1. 立刻将分区只读挂载(避免被覆盖)
mount -o remount,ro /dev/sda1
# 或卸载
umount /dev/sda1

# 2. 恢复单个文件(指定路径,相对分区根)
extundelete /dev/sda1 --restore-file path/to/file.txt

# 3. 恢复单个目录
extundelete /dev/sda1 --restore-directory path/to/dir

# 4. 恢复所有可恢复的文件
extundelete /dev/sda1 --restore-all

# 5. 按时间恢复(--after 时间戳)
extundelete /dev/sda1 --restore-all --after $(date -d "2024-01-01 00:00:00" +%s)

# 恢复的文件会放在当前目录的 RECOVERED_FILES/ 下
```

## testdisk(分区/文件系统损坏时)

```bash
# 启动交互界面
testdisk /dev/sda

# 选项流程:
# 1. Create new log file
# 2. 选择磁盘
# 3. Intel/EFI GPT 分区表类型
# 4. Analyse > Quick Search(快速搜索丢失分区)
# 5. 找到分区后 Write 写入新分区表
```

## photorec(目录结构完全丢失时)

```bash
# 按文件签名扫描整个磁盘,按类型恢复
photorec /dev/sda
# 输出到指定目录(可能上万文件,无原文件名,按类型分类)
```

## R-Linux

> 官网:<https://www.r-tt.com/free_linux_recovery/>

R-Linux 是 R-Studio 的 Linux 免费版,运行在 Windows 上,可扫描 Linux 硬盘/镜像。GUI 操作适合不熟悉命令行的用户。

## 恢复成功的关键

1. **第一时间停止写入**:发现误删后立刻 `umount` 或 `mount -o remount,ro`
2. **不要安装恢复工具到同一分区**:下载/安装工具的写入会覆盖被删数据
3. **优先恢复日志(journal)**:ext3/ext4 的 journal 保留了最近的删除记录,extundelete 默认优先利用
4. **备份镜像优先**:对重要磁盘,先用 `dd` 做完整镜像,再在镜像上恢复
   ```bash
   dd if=/dev/sda1 of=/mnt/backup/sda1.img bs=4M status=progress
   ```

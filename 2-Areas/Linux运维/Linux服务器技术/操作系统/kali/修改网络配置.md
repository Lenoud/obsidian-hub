# Kali 网络配置

Kali 基于 Debian,网络管理方式有两种:传统的 `/etc/network/interfaces` + `networking` 服务,以及现代的 NetworkManager(`nmcli` / `nmtui`)。桌面版默认使用 NetworkManager,服务器/精简安装多用前者。

## 两种管理方式对比

| 方式 | 配置入口 | 适用场景 |
| --- | --- | --- |
| 传统 interfaces | `/etc/network/interfaces` | 服务器、无桌面环境 |
| NetworkManager | `nmcli` / `nmtui` / 图形界面 | 桌面版默认 |

> 两者不能同时管理同一网卡。若 `interfaces` 中已配置某接口,NetworkManager 会标记为 **unmanaged**。

## 方式一:修改 `/etc/network/interfaces`

配置文件路径:

```text
/etc/network/interfaces
```

静态 IP 配置示例(以 eth0、`192.168.100.43/24` 为例):

```text
source /etc/network/interfaces.d/*

auto lo
iface lo inet loopback

auto eth0
iface eth0 inet static
    address 192.168.100.43
    netmask 255.255.255.0
    gateway 192.168.100.2
    dns-nameservers 8.8.8.8 114.114.114.114
```

应用配置(从 DHCP 切到静态时建议先清空旧地址,避免残留路由):

```bash
sudo ip addr flush dev eth0
sudo systemctl restart networking.service
```

## 方式二:使用 nmcli(现代推荐)

查看现有连接:

```bash
nmcli connection show
```

修改连接为静态 IP:

```bash
nmcli connection modify "Wired connection 1" \
    ipv4.addresses 192.168.100.43/24 \
    ipv4.gateway 192.168.100.2 \
    ipv4.dns "8.8.8.8 114.114.114.114" \
    ipv4.method manual
nmcli connection up "Wired connection 1"
```

切回 DHCP:

```bash
nmcli connection modify "Wired connection 1" ipv4.method auto
nmcli connection up "Wired connection 1"
```

交互式配置工具:

```bash
nmtui
```

## 验证配置

```bash
ip addr show eth0
ip route show
cat /etc/resolv.conf
```

## 常见问题

| 问题 | 解决办法 |
| --- | --- |
| 切换静态 IP 后网关残留 | 先执行 `sudo ip addr flush dev eth0` 清空旧地址再重启服务 |
| NetworkManager 显示 unmanaged | 检查 `/etc/network/interfaces` 是否托管了该接口,或查看 `/etc/NetworkManager/NetworkManager.conf` 的 `managed=true` |
| DNS 不生效 | `resolvconf` 未安装时,可手写 `/etc/resolv.conf` 或安装 `resolvconf` 包 |
| 网卡名不是 eth0 | 现代 systemd 使用 `enp3s0` / `ens33` 等预测名,用 `ip link` 确认实际名称 |

# 桌面图标存放目录

Linux 桌面环境通过 `.desktop` 文件描述应用的启动方式、图标和分类,这些文件决定了应用菜单(开始菜单)中显示哪些条目。

## .desktop 文件存放路径

| 路径 | 作用范围 | 写入权限 |
| --- | --- | --- |
| `/usr/share/applications/` | 系统全局,所有用户可见 | 需要 root |
| `~/.local/share/applications/` | 仅当前用户 | 普通用户 |
| `/usr/local/share/applications/` | 本地安装的全局应用 | 需要 root |

> 通过 `apt` / `dpkg` 安装的软件默认把 `.desktop` 放在 `/usr/share/applications/`;手动安装或绿色版软件可复制到 `~/.local/share/applications/`。

## .desktop 文件示例

以自定义应用为例,创建 `~/.local/share/applications/myapp.desktop`:

```ini
[Desktop Entry]
Type=Application
Name=My Application
Name[zh_CN]=我的应用
Comment=A custom application
Exec=/opt/myapp/run.sh
Icon=myapp
Terminal=false
Categories=Development;Utility;
StartupNotify=true
```

关键字段说明:

| 字段 | 含义 | 常见取值 |
| --- | --- | --- |
| `Type` | 条目类型 | `Application` / `Link` / `Directory` |
| `Name` | 显示名称 | 任意字符串 |
| `Exec` | 启动命令 | 可执行文件完整路径,支持 `%f`、`%F` 等占位符 |
| `Icon` | 图标 | 图标名(从 `/usr/share/icons/` 查找)或绝对路径 |
| `Terminal` | 是否在终端运行 | `true` / `false` |
| `Categories` | 菜单分类 | `Development;Network;Utility;Game;` |
| `StartupNotify` | 是否显示启动反馈 | `true` / `false` |

## 应用菜单不显示的排查

```bash
# 1. 校验 .desktop 文件语法是否正确
desktop-file-validate ~/.local/share/applications/myapp.desktop

# 2. 注册/刷新桌面数据库(手动安装的应用)
sudo update-desktop-database /usr/share/applications/

# 3. 确认 Exec 指向的可执行文件有执行权限
chmod +x /opt/myapp/run.sh
ls -l /opt/myapp/run.sh

# 4. 检查文件后缀必须是 .desktop,且 Type=Application
grep -E "Type=|Exec=" ~/.local/share/applications/myapp.desktop
```

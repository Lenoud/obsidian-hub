# 抓包工具

tcpdump的表格形式的介绍：

| 参数 | 描述 |
| --- | --- |
| -i <interface> | 指定要监听的网络接口，如eth0、en0等。 |
| -n | 使用数字格式显示IP地址和端口号，而不进行反向解析。 |
| -p | 不将网卡设置为混杂模式，以避免捕获到本机发送和接收的数据包。 |
| -s <snaplen> | 设置每个数据包的最大捕获长度（以字节为单位），默认为65535字节。 |
| -w <file> | 将捕获到的数据包写入指定的文件中。 |
| -r <file> | 读取指定文件中的数据包进行分析。 |
| -c <count> | 设置捕获数据包的数量限制。 |
| -q | 在输出时减少冗长信息。 |
| host <host> | 指定要监视的主机或IP地址。 |
| port <port> | 指定要监视的端口号。 |

使用方式：

1. 监听指定接口上的所有流量并显示详细信息：

```bash
tcpdump -i eth0
```

2. 显示某个主机的流量：

```bash
tcpdump host 192.168.0.1
```

3. 捕获指定端口的流量：

```bash
tcpdump port 80
```

4. 将捕获到的数据包写入文件：

```bash
tcpdump -w capture.pcap
```

5. 从文件中读取数据包并进行分析：

```bash
tcpdump -r capture.pcap
```

# 连通性检测

下面是关于 ping、traceroute、tracepath 和 mtr 四个命令的介绍和常用参数以及使用方式的表格形式总结：

| **命令** | **功能描述** | **常用参数** | **使用方式** |
| :---: | :---: | --- | :---: |
| **ping** | 网络连接测试工具，用于检测网络连接是否正常 | -c：指定要发送的数据包数量
-i：指定发送ICMP请求的间隔时间
-s：指定ICMP数据包的大小
-t：设置回显请求数据包的生存时间（TTL） | ping www.baidu.com

ping -c 4 -i 0.2 www.google.com |
| **traceroute** | 用于追踪数据包经过的路径，从而查找网络问题 | -I：使用ICMP回显请求代替UDP数据报
-p：指定使用的端口号
-T：使用TCPSYN数据包代替UDP数据报 | traceroute www.baidu.com

traceroute -I -p 443 www.github.com |
| **tracepath** | 与 traceroute 功能类似，但比 traceroute 更简单，并且可以批量执行 | -n：不进行反向DNS解析
-P：指定要使用的协议（支持TCP、UDP、SCTP） | tracepath www.baidu.com

tracepath -n -P tcp www.google.com |
| **mtr** | 结合了 ping 和 traceroute 的功能，可以连续跟踪路由路径和测量网络延迟 | -c：指定要发送的数据包数量
-i：指定发送ICMP请求的间隔时间
-s：指定ICMP数据包的大小
-T：使用TCPSYN数据包代替UDP数据报 | mtr www.baidu.com

mtr -c 10 -i 1 www.google.com |

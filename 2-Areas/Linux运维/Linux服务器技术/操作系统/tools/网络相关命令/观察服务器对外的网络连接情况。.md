# 观察服务器对外的网络连接情况

实时监控服务器**对外**网络流量的工具对比与用法,排查"服务器在和谁通信、走了什么流量"。

## 工具对比

| 工具 | 作用 | 工作层 | 是否实时 |
| --- | --- | --- | --- |
| **iftop** | 按连接对显示带宽 | 数据链路层(基于 host) | 实时 |
| **nethogs** | 按进程显示带宽 | 进程级 | 实时 |
| **iptraf-ng** | 多种统计(连接/LAN/协议) | 网络层 | 实时 |
| **vnstat** | 长期流量统计 | 总量 | 累计 |
| **ss / netstat** | 当前所有连接列表 | 传输层 | 快照 |
| **tcpdump** | 抓包分析 | 全栈 | 实时 |
| **tcptrack** | TCP 连接速率 | 传输层 | 实时 |

## iftop(最常用)

按"主机对"(本机 ↔ 远端 IP)显示实时带宽:

```bash
# 实时观察 eth0 的连接流量
iftop -i eth0 -n -P
# -n: 不解析主机名(快)
# -P: 显示端口
# -N: 显示端口号而不是服务名

# 常用界面快捷键(TUI 下):
#   p  切换显示端口
#   n  切换 DNS 解析
#   s  切换源地址显示
#   d  切换目的地址显示
#   t  切换显示模式(单向/双向)
#   1/2/3  按近 10s/40s/源地址排序
#   q  退出
```

输出列含义:

```text
     12.5Kb    25.0Kb     37.5Kb    50.0Kb  62.5Kb
└────────────────────────────────────────────────────
host-A.domain.com     =>   host-B.domain.com    1.20Kb  800b   1.00Kb
                      <=                        2.40Kb  1.50Kb 2.00Kb
```

最右三列分别是**近 2 秒、10 秒、40 秒**的平均速率。

## nethogs(按进程查)

找出**哪个进程**在跑流量:

```bash
nethogs
# 按 PID/USER/PROGRAM/DEV/SENT/RECEIVED 列显示

# 只看某网卡
nethogs eth0
```

## ss / netstat(连接快照)

```bash
# 所有 TCP 连接(含状态)
ss -tan | head -20
ss -tan state established | wc -l      # 已建立的连接数
ss -tan state time-wait | wc -l        # TIME_WAIT 数

# 哪些进程占用端口
ss -tlnp

# 传统 netstat(已废弃,推荐 ss)
netstat -anp | grep ESTABLISHED
```

## vnstat(长期流量统计)

```bash
apt install vnstat
systemctl enable --now vnstat

# 等待几小时/几天后查看
vnstat                    # 总览(按天/月)
vnstat -h                 # 小时统计
vnstat -d                 # 日统计
vnstat -l                 # 实时(rx/tx)
```

## 实战场景

### 场景 1:突然带宽跑满,谁干的?

```bash
# 1. 看哪个网卡满
ip -s link

# 2. 看哪个进程在发数据
nethogs

# 3. 看本机和哪个 IP 通信最频繁
iftop -i eth0 -n

# 4. 锁定后,查那个 IP 是谁
ss -tan | grep <ip>
```

### 场景 2:追踪某个进程的网络行为

```bash
# 1. 找进程 PID
pidof nginx
# 12345

# 2. 查看该进程的所有连接
lsof -p 12345 -i

# 3. 抓包
tcpdump -i any -w nginx.pcap 'tcp and host 1.2.3.4'
```

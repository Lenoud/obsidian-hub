# 网络管理

网络管理相关技术笔记。

## 一．实训概要
了解网络接口配置与管理的作用，掌握通过静态方式配置网络接口、设置主机名及验证网络功能的方法。

---

## 二．实训内容
1. **查看网络接口和IP地址：** 检查当前系统的网络接口配置和IP地址信息。
2. **更改为静态IP：**
    - 将网络接口配置从DHCP改为静态获取IP地址。
    - 手动设置IP地址、子网掩码、网关及DNS服务器。
3. **测试网络配置：** 验证配置是否成功。
4. **配置主机名：** 修改系统主机名为自己的姓名拼音，并验证更改是否生效。

---

## 三．实训过程
### 1. 查看当前系统的网络接口配置和IP地址
**目标：** 查看当前网络接口配置及获取的IP地址。
**操作步骤：**

1. 使用以下命令查看网络接口和IP地址：

```bash
ip addr show
```

2. 记录需要修改的网络接口名称及当前的网络配置（如`ens33`）。

---

### 2. 更改为静态IP地址
**目标：** 将网络接口从DHCP改为静态获取IP地址，并手动设置参数。

#### 方法1：使用命令行配置
**操作步骤：**

1. 打开网络配置文件（以`ens33`为例）：

```bash
vi /etc/sysconfig/network-scripts/ifcfg-ens33
```

2. 修改以下内容：

```bash
BOOTPROTO="static"
IPADDR=192.168.1.<学号后两位或+100>
NETMASK=255.255.255.0  # 子网掩码
GATEWAY=192.168.1.1    # 网关
DNS1=8.8.8.8
DNS2=114.114.114.114
ONBOOT="yes"
```

3. 保存并退出（`wq`）。
4. 重启网络服务使配置生效：

```bash
systemctl restart network
```

#### 方法2：使用图形界面配置（适用于图形化环境）
1. 打开“网络设置”界面。
2. 选择需要更改的网络接口，点击“设置”。
3. 在IPv4设置中选择“手动”，填写以下参数：
    - IP地址：`192.168.1.<学号后两位或+100>`
    - 子网掩码：`255.255.255.0`
    - 网关：`192.168.1.1`
    - DNS服务器：`8.8.8.8` 和 `114.114.114.114`
4. 保存配置并应用更改。

---

### 3. 测试网络配置是否成功
**目标：** 确认静态IP配置是否生效，测试网络连接是否正常。

**操作步骤：**

1. 检查当前网络接口的IP地址：

```bash
ip addr show
```

确保显示的IP地址与静态配置一致。

2. 测试网络连通性：

```bash
ping -c 4 8.8.8.8
ping -c 4 www.baidu.com
```

确保能够正常通信。

---

### 4. 配置系统主机名
**目标：** 修改系统主机名为自己的姓名拼音，并验证是否生效。

**操作步骤：**

1. 临时修改主机名（立即生效，但重启后失效）：

```bash
hostnamectl set-hostname <姓名拼音>
```

例如：

```bash
hostnamectl set-hostname zhangsan
```

2. 永久修改主机名（重启后生效）：
    - 修改`/etc/hostname`文件：

```bash
echo "<姓名拼音>" > /etc/hostname
```

    - 修改`/etc/hosts`文件，添加主机名与IP地址的映射（可选）：

```bash
192.168.1.<IP地址> <姓名拼音>
```

3. 验证主机名更改是否生效：

```bash
hostname
```

---

## 四．验证与总结
1. **验证网络配置：**
    - 使用`ip addr`命令检查静态IP配置是否正确。
    - 使用`ping`命令测试网络连通性。
2. **验证主机名：**
    - 使用`hostname`命令确认主机名是否已更新为姓名拼音。

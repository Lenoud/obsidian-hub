# IPv6

IPv6 地址结构、地址分类、与 IPv4 的差异,以及常用配置和测试方法。

## IPv6 地址结构

IPv6 地址长度 **128 位**,通常以 8 组 16 位十六进制数表示,用冒号分隔:

```text
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

简写规则:
- 每组前导 0 可省略(`0db8` → `db8`)
- 连续全 0 组可用 `::` 表示一次(整个地址只能用一次)

例:`2001:db8:85a3::8a2e:370:7334`

## 地址分类

| 类型 | 前缀 | 说明 |
| --- | --- | --- |
| 全球单播(Global Unicast) | `2000::/3` | 公网可路由,相当于 IPv4 公网 IP |
| 链路本地(Link-Local) | `fe80::/10` | 仅本网段有效,自动生成,用于邻居发现 |
| 唯一本地(Unique Local) | `fc00::/7` | 内网,相当于 IPv4 私网地址 |
| 回环(Loopback) | `::1/128` | 等价于 IPv4 的 `127.0.0.1` |
| 多播(Multicast) | `ff00::/8` | 等价于 IPv4 的 D 类地址 |
| 任播(Anycast) | — | 一对最近,机制特殊 |

## 常用命令

```bash
# 查看 IPv6 地址
ip -6 addr show

# 查看 IPv6 路由
ip -6 route show

# ping IPv6
ping6 2001:4860:4860::8888

# 端口连通性测试(以 9182 为例)
curl -v "http://[2408:8352:213:8b3:b538:fa0e:b8e7:7c91]:9182"
```

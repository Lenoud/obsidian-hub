# kubeadm 搭建 K8s 集群

使用 kubeadm 在 CentOS / Ubuntu 上快速搭建 Kubernetes 集群。

> **版本说明**:本文档原版本为 K8s 1.18.0(2020 年发布,已 EOL),已升级为通用流程,实际安装请使用当前 LTS 版本(1.28 / 1.30)。如需特定版本,把命令里的版本号改掉即可。

## 环境规划

| 节点 | IP | 角色 | 配置 |
| --- | --- | --- | --- |
| k8s-master | 192.168.100.38 | control-plane | 2C4G+ |
| k8s-node1 | 192.168.100.39 | worker | 2C4G+ |
| k8s-node2 | 192.168.100.40 | worker | 2C4G+ |

## 一、每台主机都要执行(初始化)

```bash
# 关闭防火墙
systemctl stop firewalld && systemctl disable firewalld

# Ubuntu 用 ufw
# ufw disable

# 关闭 SELinux(CentOS / RHEL)
setenforce 0
sed -i 's/enforcing/disabled/' /etc/selinux/config

# 关闭 swap(kubeadm 要求)
swapoff -a
sed -ri 's/.*swap.*/#&/' /etc/fstab

# 设置主机名(每台不同)
hostnamectl set-hostname k8s-master        # 在 master 上
hostnamectl set-hostname k8s-node1         # 在 node1 上

# 在 master 配置 hosts
cat >> /etc/hosts << EOF
192.168.100.38 k8s-master
192.168.100.39 k8s-node1
192.168.100.40 k8s-node2
EOF

# 桥接流量透传给 iptables
cat > /etc/sysctl.d/k8s.conf << EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
EOF
sysctl --system

# 时间同步
yum install -y chrony          # CentOS
# apt install -y chrony        # Ubuntu
systemctl enable --now chronyd
```

## 二、安装容器运行时(containerd)

K8s 1.24+ 移除了 Docker Shim,**必须用 containerd**(或其他兼容 CRI 的运行时):

```bash
# CentOS
yum install -y yum-utils
yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
yum install -y containerd.io

# Ubuntu
# apt install -y containerd

# 生成默认配置
containerd config default > /etc/containerd/config.toml

# 配置 SystemdCgroup(K8s 1.25+ 要求)
sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml

# 配置 sandbox 镜像(避免国内拉不下 pause 镜像)
sed -i 's|registry.k8s.io/pause:3.[0-9]*|registry.aliyuncs.com/google_containers/pause:3.9|' \
    /etc/containerd/config.toml

systemctl enable --now containerd
```

## 三、安装 kubeadm / kubelet / kubectl

```bash
# 配置 K8s 软件源(阿里云镜像)
cat > /etc/yum.repos.d/kubernetes.repo << EOF
[kubernetes]
name=Kubernetes
baseurl=https://mirrors.aliyun.com/kubernetes/yum/repos/kubernetes-el7-x86_64
enabled=1
gpgcheck=0
repo_gpgcheck=0
EOF

# 安装指定版本(以 1.28 为例)
yum install -y kubelet-1.28.0 kubeadm-1.28.0 kubectl-1.28.0
systemctl enable kubelet
```

> Ubuntu 用 `apt install -y kubelet=1.28.0-00 kubeadm=1.28.0-00 kubectl=1.28.0-00`,需先加 K8s apt 源。

## 四、master 节点初始化

```bash
# 拉取依赖镜像(用国内源)
kubeadm config images pull \
  --image-repository registry.aliyuncs.com/google_containers \
  --kubernetes-version v1.28.0

# 初始化集群
kubeadm init \
  --apiserver-advertise-address=192.168.100.38 \
  --image-repository registry.aliyuncs.com/google_containers \
  --kubernetes-version v1.28.0 \
  --service-cidr=10.96.0.0/12 \
  --pod-network-cidr=172.20.0.0/16

# 配置 kubectl(按提示执行)
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

# 验证
kubectl get nodes
# 此时 master 是 NotReady,因为还没装网络插件
```

## 五、worker 节点加入

在 master init 完成时,会输出 join 命令(含 token):

```bash
# 在每个 worker 节点执行
kubeadm join 192.168.100.38:6443 \
  --token <token> \
  --discovery-token-ca-cert-hash sha256:<hash>

# 如果 token 过期(默认 24h),在 master 重新生成
kubeadm token create --print-join-command
```

## 六、部署网络插件(Flannel)

```bash
# 下载 Flannel 配置(用最新稳定版)
wget https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml

# 注意 pod-network-cidr 必须与 init 时一致(本文用 172.20.0.0/16)
# 如果 kube-flannel.yml 里默认是 10.244.0.0/16,改成 172.20.0.0/16

kubectl apply -f kube-flannel.yml

# 验证(等 1-2 分钟,所有节点变 Ready)
kubectl get nodes
kubectl get pods -n kube-flannel
```

## 七、常见问题与解决

### 1. 节点一直是 NotReady

```bash
# 看 kubelet 日志
journalctl -u kubelet -f

# 常见原因:
# - containerd 没启动
# - SystemdCgroup 没配置
# - CNI 插件缺失(见下)
```

### 2. CNI plugins 缺失

```bash
# 下载 cni-plugins
wget https://github.com/containernetworking/plugins/releases/download/v1.3.0/cni-plugins-linux-amd64-v1.3.0.tgz
mkdir -p /opt/cni/bin
tar zxvf cni-plugins-linux-amd64-v1.3.0.tgz -C /opt/cni/bin
```

### 3. 删除 master 污点(允许调度 Pod 到 master)

```bash
# 查看污点
kubectl get nodes -o custom-columns=NAME:.metadata.name,TAINTS:.spec.taints

# 删除
kubectl taint nodes k8s-master node-role.kubernetes.io/control-plane-
```

### 4. kubectl 自动补全

```bash
yum install -y bash-completion
source <(kubectl completion bash)
echo "source <(kubectl completion bash)" >> ~/.bashrc
```

### 5. 节点维护(驱逐 + 删除)

```bash
# 安全驱逐节点上的 Pod
kubectl drain k8s-node1 --delete-emptydir-data --force --ignore-daemonsets

# 删除节点
kubectl delete node k8s-node1

# 在被删除节点上重置
kubeadm reset
```

## 测试

```bash
# 部署 nginx
kubectl create deployment nginx --image=nginx
kubectl expose deployment nginx --port=80 --type=NodePort

# 查看 NodePort
kubectl get svc nginx
# nginx  NodePort  10.106.x.x  80:31234/TCP   ← 浏览器访问 http://<任意节点IP>:31234
```

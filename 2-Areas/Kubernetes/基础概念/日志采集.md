# 日志采集

日志采集相关技术笔记。

基于 Elasticsearch + Fluentd + Kibana（EFK）的 Kubernetes 集群日志采集方案，使用 Helm 部署 Elasticsearch 集群（master/data/client）、Kibana 可视化面板和 Fluentd 采集器。

## 说明

| 组件 | 版本 | 用途 |
| :--- | :--- | :--- |
| Elasticsearch | 7.17.3 | 日志存储与搜索引擎 |
| Kibana | 7.17.3 | 日志可视化面板 |
| Fluentd | v3.4.0 | 日志采集与转发 |

## 生成证书

```bash
nerdctl run --name elastic-certs --namespace k8s.io -v ./elastic-certs:/app -i -w /app elasticsearch:7.17.3 /bin/sh -c "elasticsearch-certutil ca --out /app/elastic-stack-ca.p12 --pass '' && elasticsearch-certutil cert --name security-master --dns security-master --ca /app/elastic-stack-ca.p12 --pass '' --ca-pass '' --out /app/elastic-certificates.p12"
```

## Helm 部署 Elasticsearch

```bash
helm upgrade --install es-master -f values-master.yaml -n logging .
helm upgrade --install es-data -f values-data.yaml -n logging .
helm upgrade --install es-client -f values-client.yaml -n logging .
```

## Helm 部署 Kibana

```bash
helm install kibana -f values-prod.yaml -n logging .
```

## Fluentd 配置与部署

使用 ConfigMap 管理 Fluentd 配置，DaemonSet 方式部署到每个节点采集容器日志。

> 被采集的 Pod 必须具有 `logging: "true"` 标签才会被 Fluentd 采集。

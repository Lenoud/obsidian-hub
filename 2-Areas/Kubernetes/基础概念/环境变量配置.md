# K8s 环境变量配置(env)

通过 Pod 的 `spec.containers[].env` 字段为容器注入环境变量,等价于 Docker 的 `-e`。

## 静态环境变量

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: env-demo
spec:
  containers:
    - name: app
      image: busybox
      command: ["sh", "-c", "echo $hello && sleep 3600"]
      env:
        - name: hello
          value: "nihao"
        - name: hello1
          value: "nihao1"
```

## 引用其他资源

### 引用 ConfigMap

```yaml
env:
  - name: DB_HOST
    valueFrom:
      configMapKeyRef:
        name: app-config
        key: db_host
```

### 引用 Secret

```yaml
env:
  - name: DB_PASSWORD
    valueFrom:
      secretKeyRef:
        name: db-secret
        key: password
```

### 引用 Pod 字段 downward API

```yaml
env:
  - name: POD_NAME
    valueFrom:
      fieldRef:
        fieldPath: metadata.name
  - name: NODE_NAME
    valueFrom:
      fieldRef:
        fieldPath: spec.nodeName
```

## 一次性注入所有键值(envFrom)

```yaml
envFrom:
  - configMapRef:
      name: app-config          # 把 app-config 里所有 key 都注入为环境变量
  - secretRef:
      name: app-secret
```

## 常见问题

| 问题 | 原因 |
| --- | --- |
| 容器内 `echo $hello` 为空 | 改 env 后没重新创建 Pod(只 restart 无效,要 `kubectl delete pod`) |
| value 必须是字符串 | YAML 中 `value: 123` 会被解析成 int,需 `value: "123"` |
| 引用 ConfigMap 不存在 | 默认会阻塞 Pod 启动,加 `optional: true` 可允许缺失 |

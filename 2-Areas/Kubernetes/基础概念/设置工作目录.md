# K8s 设置工作目录(workingDir)

通过 Pod 的 `spec.containers[].workingDir` 字段指定容器启动后的工作目录,等价于 Docker 的 `--workdir` / Dockerfile 的 `WORKDIR`。

## 示例

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: workingdir-demo
spec:
  containers:
    - name: app
      image: busybox
      command: ["sh", "-c", "pwd && ls"]     # 输出会显示 /app
      workingDir: /app                       # 关键字段
      volumeMounts:
        - name: app-data
          mountPath: /app
  volumes:
    - name: app-data
      emptyDir: {}
```

## 注意事项

| 注意点 | 说明 |
| --- | --- |
| 目录必须存在 | 容器内若不存在该路径,启动时会报错 `chdir: no such file or directory` |
| 优先级低于 `command` 的绝对路径 | `workingDir` 影响相对路径解析,不影响绝对路径 |
| 与 ENTRYPOINT/CMD 配合 | `workingDir` 会覆盖镜像里的 WORKDIR |
| 每个 container 独立 | 多容器 Pod 中各 container 可设置不同 workingDir |

## 相关命令

```bash
# 进入容器后查看当前工作目录
kubectl exec -it workingdir-demo -- pwd

# 默认(未设 workingDir)使用镜像里的 WORKDIR
# 例如 nginx:alpine 默认 /,busybox 默认 /
```

# 网络调试容器

包含 BusyBox 和 Netshoot 两个网络调试工具容器，用于在 Docker 网络中进行连通性测试和网络诊断。

## 部署

```yaml
version: "3.8"
services:
  busybox:
    hostname: busybox1
    container_name: busybox1
    image: busybox:1.37.0 # 使用指定版本的 BusyBox 镜像
    restart: unless-stopped # 自动重启策略，除非手动停止
    tty: true # 分配一个伪终端
    networks:
      - db_bridge # 连接到外部定义的 db_bridge 网络
    command: sh # 启动一个交互式 shell

  netshoot:
    hostname: netshoot1
    container_name: netshoot1
    image: nicolaka/netshoot:v0.13
    restart: unless-stopped
    tty: true
    networks:
      - db_bridge
    command: /bin/bash
networks:
  db_bridge:
    external: true
```

## 说明

- **BusyBox**（busybox1）：轻量级 Linux 工具集，提供基本的 shell 和网络命令
- **Netshoot**（netshoot1）：专业的网络诊断工具集，包含 tcpdump、nslookup、curl、nmap 等工具
- 两个容器均连接到外部网络 `db_bridge`
- `tty: true`：分配伪终端，支持交互式操作
- 使用 `docker exec -it busybox1 sh` 或 `docker exec -it netshoot1 /bin/bash` 进入容器

# Containerd 配置登录私有仓库

Containerd 配置登录私有仓库相关技术笔记。

介绍如何通过 `nerdctl` 和 `ctr` 两种方式配置 Containerd 登录并推送镜像到 Harbor 私有仓库。

## 说明

| 工具 | 适用场景 | 认证方式 |
| :--- | :--- | :--- |
| nerdctl | 单机推送/拉取镜像 | `nerdctl login` |
| ctr | 配置 `config.toml` 实现集群级别镜像同步 | 配置文件 + `--user` 参数 |

## nerdctl 推送（HTTP 方式）

```bash
# 创建 hosts.toml 配置目录
mkdir -p /etc/containerd/certs.d/192.168.100.105:80
```

创建 `hosts.toml` 文件：

```toml
server = "https://docker.io"

[host."http://192.168.100.105"]
  capabilities = ["pull", "resolve","push"]
```

登录并推送：

```bash
echo Harbor12345 | nerdctl login --username "admin" --password-stdin 192.168.100.40
nerdctl login --username "admin" --password "Harbor12345" 192.168.100.105
```

## ctr 推送

集群所有节点修改 Containerd 配置，添加 Harbor 仓库的认证信息：

```bash
# vi /etc/containerd/config.toml
```

```toml
[plugins."io.containerd.grpc.v1.cri".registry.mirrors] # 此字段后加入以下配置
  [plugins."io.containerd.grpc.v1.cri".registry.mirrors."harbor.com"]
    endpoint = ["http://harbor.com"]
  [plugins."io.containerd.grpc.v1.cri".registry.mirrors."10.24.2.200"]
    endpoint = ["http://10.24.2.200"]
```

重启 Containerd 以生效配置：

```bash
systemctl daemon-reload
systemctl restart containerd

# 标记并推送镜像
ctr -n k8s.io images tag docker.io/library/maven:3.8.2-openjdk-8 10.24.2.200/library/maven:3.8.2-openjdk-8
ctr -n k8s.io images push --plain-http=true --user admin:Harbor12345 10.24.2.200/library/maven:3.8.2-openjdk-8
```

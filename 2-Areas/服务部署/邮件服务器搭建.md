# 邮件服务器搭建

自建邮件服务器的常见方案与注意事项。邮件服务因涉及 DNS、反垃圾、端口封锁等,部署难度较高,通常建议优先使用 SaaS 服务。

## 方案对比

| 方案 | 类型 | 适用场景 | 备注 |
| --- | --- | --- | --- |
| **Mailu** | 自建全套(Docker) | 个人/小团队,需要完整收发 | 集成 IMAP/SMTP/Webmail/Antispam |
| **Postfix/Dovecot** | 自建基础 | 已有服务器,只需 MTA | 配置繁琐,需自行对接反垃圾 |
| **Postfix + 外部 SMTP 中继** | 混合 | 国内服务器 25 端口被封 | 通过 BaoTa / SendGrid 等中继 |
| **SendGrid / MailGun / SES** | SaaS | 业务邮件(通知/营销) | 无需运维,按量付费 |
| **腾讯企业邮 / 阿里邮箱** | SaaS | 企业正式邮箱 | 国内可达性最好 |

## Mailu(Docker 全套)

> 仓库:<https://github.com/Mailu/Mailu>

Mailu 是开源的邮件服务套件,提供 IMAP / SMTP / Webmail / Antispam / Antivirus 一站式方案,适合 Docker 部署。

### 准备工作

- 一个**独立公网 IP**(不能是主要云厂商的默认 IP 段,容易被列入黑名单)
- 一个域名,并能修改 DNS
- 开放端口:25(SMTP)、465(SMPTS)、587(Submission)、993(IMAPS)
- **国内服务器:25 端口通常被封锁**,需提交工单解封或改用外部 SMTP 中继

### 必要的 DNS 记录

```text
# A 记录:邮件服务器主机名
mail.example.com.   IN  A     1.2.3.4

# MX 记录:指向邮件服务器主机名
example.com.        IN  MX    10 mail.example.com.

# SPF:授权哪些 IP 可以发信
example.com.        IN  TXT   "v=spf1 mx a -all"

# DKIM:Mailu 生成后给出(选 择 mail._domainkey)
mail._domainkey.example.com. IN TXT "v=DKIM1; k=rsa; p=MIGfMA0..."

# DMARC:告诉接收方如何处理未通过 SPF/DKIM 的邮件
_dmarc.example.com. IN  TXT   "v=DMARC1; p=quarantine; rua=mailto:postmaster@example.com"
```

### 部署步骤(精简版)

```bash
# 1. 使用官方向导生成 docker-compose.yml 与 env
#    https://setup.mailu.io/

# 2. 下载生成的配置
wget -O mailu.zip <从向导获取的链接>
unzip mailu.zip -d mailu && cd mailu

# 3. 启动
docker compose -f docker-compose.yml up -d

# 4. 创建管理员账号
docker compose exec admin flask mailu-admin admin example.com
```

## 常见问题

| 问题 | 原因 | 解决 |
| --- | --- | --- |
| 发往 Gmail/QQ 进垃圾箱 | SPF/DKIM/DMARC 未配置或不正确 | 三件套全配齐,等待 DNS 生效(24h+) |
| 25 端口超时 | 国内运营商/云厂商封锁 | 提工单解封,或用外部 SMTP 中继 |
| 收不到外部邮件 | MX 记录错误或未生效 | 用 `dig MX example.com` 验证 |
| IP 在黑名单 | IP 信誉低 | 查询 Spamhaus 等,申请解封 |

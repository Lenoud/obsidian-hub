# 字符串变量：格式介绍

Shell 字符串变量的格式介绍（单引号、双引号、无引号）。

### 字符串的三种格式区别
#### 使用单引号''的字符串
任何字符都会原样输出，在拼接字符串中使用变量是无效的。

```text
[root@rockylinux-docker shell]# var1=liubiao

[root@rockylinux-docker shell]# echo '$var1 hello!'
$var1 hello!

#原样输出了变量名字，没有识别到变量
```

#### 使用双引号""的字符串
其中包含了变量，那么该变量会被解析得到值，而不是原样输出。

字符串中还可以出现双引号的子字符串，但需要转义。

```text
[root@rockylinux-docker shell]# var2="${var1} hello  world! \"转义\""

[root@rockylinux-docker shell]# echo ${var2}
liubiao hello world! "转义"

```

#### 不用引号包围的字符串
不被引号包围的字符串中出现变量时也会被解析。这一点和双引号包围的字符串一样。

字符串中不能出现空格，否则空格后边的字符串会作为其他命令解析。

```text
[root@rockylinux-docker shell]# var3=${var1}helloworld!
[root@rockylinux-docker shell]# echo ${var3}
liubiaohelloworld!

[root@rockylinux-docker shell]# var3=${var1} helloworld!
-bash: helloworld!: command not found

#提示没有找到对应的命令
```

### 获取字符串的长度
#### 语法
```text
${#变量名}
```

#### 示例
```text
[root@rockylinux-docker shell]# var1="123456789"

[root@rockylinux-docker shell]# echo ${#var1}
9

```

# Shell 重定向：输入输出

本文介绍 Shell 中标准输入、标准输出和标准错误的重定向操作，包括覆盖写入、追加重定向和错误输出的用法。

## 什么是重定向

重定向是指将命令的默认输入输出方向进行改变，将数据流重新指向文件或其他设备。

### 标准输入输出
| 文件描述符 | 设备文件 | 说明 |
| --- | --- | --- |
| 0 | stdin | 标准输入，默认从键盘读取 |
| 1 | stdout | 标准输出，默认输出到屏幕 |
| 2 | stderr | 标准错误，默认输出到屏幕 |

### 输出重定向

| 符号 | 作用 | 示例 |
| --- | --- | --- |
| > | 覆盖写入 | echo "hello" > file.txt |
| >> | 追加写入 | echo "hello" >> file.txt |
| 2> | 错误覆盖 | command 2> error.log |
| 2>> | 错误追加 | command 2>> error.log |
| &> | 全部覆盖 | command &> all.log |
| &>> | 全部追加 | command &>> all.log |

### 输入重定向

| 符号 | 作用 | 示例 |
| --- | --- | --- |
| < | 从文件读取输入 | command < input.txt |
| << | Here Document | command << EOF |

# 通过 SSH 执行远程命令时，无法自动加载环境变量问题

解决通过 SSH 执行远程命令时无法自动加载环境变量的问题。

## 问题场景
目前我的环境变量都存储在 `/etc/profile` 文件中，当我通过远程 SSH 执行一些命令时，提示命令找不到，如下所示：

![[1728905067017-113424b5-873b-4d58-9529-2e8f1a08d2d1.png]]

## 问题出现原因
这里找到了一张出自尚硅谷的图片，很好的解释了该问题：

![[1728905067041-35456c65-bdd3-497a-9fff-33acf7440083.png]]

这是由于 Linux 登录方式不同造成所加载的环境变量也有所不同的原因导致的。

## 解决方法
1. 创建一个环境变量文件，存储在 `/etc/profile.d/` 目录下，这样无论是否登录，都会自动加载。
2. 或者在执行命令时，通过手动加载环境变量，如下所示：

![[1728905067119-de98d7f2-17c2-4746-b771-9820bad4acad.png]]

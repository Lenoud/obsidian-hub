# Shell 运算符:布尔运算符

Shell 中的布尔运算符(`!`、`-a`、`-o`)用于组合多个条件表达式,必须放在 `[ ]` 或 `test` 命令中使用(**不能**用在 `[[ ]]` 里,后者有自己的 `&&` / `||`)。

> 推荐用 `[[ ]]` + `&&` / `||`,只在 `[ ]` 或 `test` 里用 `-a` / `-o`。

## 运算符表

| 运算符 | 说明 | 示例 |
| --- | --- | --- |
| `!` | 逻辑非(取反) | `[ ! -f /tmp/x ]`(文件不存在时返回 0) |
| `-a` | 逻辑与(AND) | `[ -f /tmp/x -a -r /tmp/x ]`(存在且可读时返回 0) |
| `-o` | 逻辑或(OR) | `[ -f /tmp/x -o -f /tmp/y ]`(任一存在返回 0) |

## 与 `[[ ]]` 的 `&&` / `||` 对比

```bash
# [ ] 里的 -a / -o(老写法,可移植性好)
if [ "$a" -gt 5 -a "$a" -lt 10 ]; then ...     # a 在 5~10 之间

# [[ ]] 里的 && / ||(Bash/Ksh 扩展,推荐)
if [[ $a -gt 5 && $a -lt 10 ]]; then ...
```

`[[ ]]` 优势:
- 支持模式匹配(`[[ $name == abc* ]]`)
- 支持 `=~` 正则(`[[ $ip =~ ^[0-9.]+$ ]]`)
- 变量无需引号(避免空字符串引起的语法错误)

## 易错点

| 错误写法 | 问题 | 正确写法 |
| --- | --- | --- |
| `[ 1 == 1 -a 2 == 2 ]` | `==` 是字符串比较,不该和 `-a` 混用 | `[ 1 -eq 1 -a 2 -eq 2 ]`(整数) |
| `[ $a -gt 5 -a $a -lt 10 ]`(a 为空) | 变量空时语法错 | `[[ $a -gt 5 && $a -lt 10 ]]` 或加引号 `[ "$a" -gt 5 -a "$a" -lt 10 ]` |
| `[[ $x -a $y ]]` | `[[ ]]` 里不能用 `-a` | `[[ $x && $y ]]` |

## 短路求值

`-a` / `-o` / `&&` / `||` 都是**短路求值**:
- `-a`(`&&`):第一个为假就不再判断第二个
- `-o`(`||`):第一个为真就不再判断第二个

```bash
# 利用短路设置默认值
[ -z "$VAR" ] && VAR="default"

# 等价
VAR=${VAR:-default}
```

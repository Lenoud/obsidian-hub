# 特殊变量

Shell 特殊变量（$0、$1、$#、$@、$? 等）说明。

### 目标
能说出常用的特殊变量有哪些

###  特殊变量：$n
```shell
$n

$0 获取当前shell脚本文件名字
$1~$9 代表获取第一行到第九个输入参数
第10个以上的输入参数获取参数的格式：${数字},否则无法获取(带花括号)
```

#### 执行脚本文件传入参数语法
```shell
sh 脚本文件 输入参数1  输入参数2 .....
```

#### 示例
```shell
#!/bin/bash
echo "第1个传入的参数为：${1}"
echo "第2个传入的参数为：${2}"
echo "第3个传入的参数为：${3}"
echo "第4个传入的参数为：${4}"
echo "第5个传入的参数为：${10}"

[root@rockylinux-docker shell]# bash read_1-9.sh  test1 test2 test3 test4 test5 test6 test7 test8  test9 test1000
第1个传入的参数为：test1
第2个传入的参数为：test2
第3个传入的参数为：test3
第4个传入的参数为：test4
第5个传入的参数为：test1000

```

###  特殊变量：$#
#### 作用
获取所有输入参数的个数

#### 示例
```shell
[root@rockylinux-docker shell]# cat read_1-9.sh
#!/bin/bash
echo "第1个传入的参数为：${1}"
echo "第2个传入的参数为：${2}"
echo "第3个传入的参数为：${3}"
echo "第4个传入的参数为：${4}"
echo "第5个传入的参数为：${10}"

echo "所有输入参数个数：$#"

[root@rockylinux-docker shell]# bash read_1-9.sh  test1 test2 test3 test4 test5 test6 test7 test8  test9 test1000
第1个传入的参数为：test1
第2个传入的参数为：test2
第3个传入的参数为：test3
第4个传入的参数为：test4
第5个传入的参数为：test1000
所有输入参数个数：10

```

### 特殊变量：$*  $@
#### 语法
```shell
$*
$@
#含义都是获取所有输入参数，用于以后输入所有参数
```

#### $*  $@的区别
 不使用双引号括起来，功能一样

$* 和$@获取所有输入参数，格式为： $1 $2 .... $n

（ $* 和 $@ 的表现是一样的，它们会将所有参数展开为一个用空格分隔的字符串。  ）

使用双引号括起来

"$*"  获取的所有参数拼接为一个字符串 格式为： "$1 $2 .... $n"

"$@"获取一组参数列表对象，格式为："$1" "$2" ...... "$n"

使用循环打印所有输入参数可以看出区别

#### 关键区别总结
| **表达式** | **展开方式** | **行为特点** |
| --- | --- | --- |
| `$*` | 一个整体 | 将所有参数合并为一个字符串，用 IFS 分隔（ 默认情况下，IFS 的值是空格  ） |
| `$@` | 独立参数 | 将每个参数独立保留，逐一展开 |
| `"$*"` | 一个整体 | 参数合并成一个字符串，参数之间由 IFS 分隔 |
| `"$@"` | 独立参数 | 每个参数作为独立的字符串，并保持独立性（推荐用在脚本中传递参数时） |

#### 循环语法
```shell
for var in 列表变量
do   #循环开始
   命令 #循环体
done  #循环结束
```

#### 示例
可以看到@是一个个输出的

```shell
[root@rockylinux-docker shell]# cat read_1-9.sh
#!/bin/bash
echo "所有输入参数个数：$#"
echo "*号直接输出:"${*}
echo "@号直接输出"${@}

echo '循环输出@'
for var in "$@"
do   #循环开始
   echo "输出：${var}" #循环体

echo '循环输出*'
for var in "$*"
do   #循环开始
   echo "输出：${var}" #循环体
done  #循环结束

[root@rockylinux-docker shell]# bash read_1-9.sh  test1 test2 test3 test4 test5 test6 test7 test8  test9 test1000

*号直接输出:test1 test2 test3 test4 test5 test6 test7 test8 test9 test1000
@号直接输出test1 test2 test3 test4 test5 test6 test7 test8 test9 test1000
循环输出@
输出：test1
输出：test2
输出：test3
输出：test4
输出：test5
输出：test6
输出：test7
输出：test8
输出：test9
输出：test1000
循环输出*
输出：test1 test2 test3 test4 test5 test6 test7 test8 test9 test1000

```

### **特殊变量：$？**
#### 语法
```shell
$?
```

#### 含义
获取上一个shell命令的退出状态码，或者是函数的返回值

每个shell 命令的执行都有一个返回值，这个返回值用于说明命令执行是否成功

一般来说，返回的是0代表成功，非0代表执行失败

#### 示例
```shell
[root@rockylinux-docker shell]# echo $?
0
[root@rockylinux-docker shell]# asdfa
-bash: asdfa: command not found
[root@rockylinux-docker shell]# echo $?
127

```

### **特殊变量：$$**
#### 语法
```shell
$$
```

#### 作用
获取当前shell环境的id号

#### 示例
```shell
root      190219  0.0  0.0   5920  4736 pts/0    Ss   12:42   0:00 -bash

[root@rockylinux-docker shell]# echo $$
190219
```

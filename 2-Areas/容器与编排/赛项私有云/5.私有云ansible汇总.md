### 【题目1】Mariadb高可用集群
使用OpenStack私有云平台，创建4台系统为centos7.5的云主机，其中一台作为Ansible的母机并命名为ansible，另外三台云主机命名为node1、node2、node3，通过http://<IP>/ansible.tar.gz软件包在ansible节点安装Ansible服务；使用这一台母机，编写Ansible脚本（在/root目录下创建example目录作为Ansible工作目录，部署的入口文件命名为cscc_install.yaml），对其他三台云主机进行安装高可用数据库集群（MariaDB_Galera_cluster，数据库密码设置为123456）的操作（可以使用http://<IP>/gpmall-single.tar.gz中的gpmall-repo目录作为yum源安装数据库服务）。（考试系统会连接到你的Ansible节点，去执行Ansible脚本，请准备好Ansible运行环境，以便考试系统访问）

二、ansible部署mariadb_galera_cluster集群

1.创建工作目录，并生成角色目录

[root@ansible ~]# mkdir -p /root/example/roles

[root@ansible ~]# cd /root/example/roles/

[root@ansible roles]# ansible-galaxy init mariadb   生成一个mariadb角色，并且角色下的目录自动生成

- Role mariadb was created successfully

[root@ansible roles]# ls mariadb/

defaults  files  handlers  meta  README.md  tasks  templates  tests  vars

2.编写local.repo

[root@ansible roles]# cp -r /opt/mariadb-repo/ /root/example/roles/mariadb/files/

[root@ansible roles]# cat /root/example/roles/mariadb/files/local.repo

[mariadb]

name=mariadb

baseurl=file:///opt/mariadb-repo

gpgcheck=0

enabled=1

3.编写yum.yml

[root@ansible roles]# cd mariadb/tasks/

[root@ansible tasks]# cat yum.yml

- name: rm old repo

  shell: rm -rf /etc/yum.repos.d/*

- name: copy mariadb-repo

  copy:

    src: mariadb-repo

    dest: /opt/

- name: copy local.repo

  copy:

    src: local.repo

    dest: /etc/yum.repos.d/

4.编写install.yml

[root@ansible tasks]# cat install.yml

- name: node1-3 install packages

  yum:

    name:

      - MariaDB-server

      - MariaDB-client

      - galera

    state: present

5.编写init.yml

[root@ansible tasks]# cat init.yml

- name: node1-3 start mariadb

  service:

    name: mariadb

    state: started

- name: node1-3 set database'password

  shell: mysqladmin -uroot password 123456

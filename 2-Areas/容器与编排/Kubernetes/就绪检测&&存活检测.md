# 就绪检测与存活检测

就绪检测与存活检测相关技术笔记。

Kubernetes 通过 `livenessProbe`（存活检测）和 `readinessProbe`（就绪检测）来监控容器的健康状态，支持 HTTP、TCP 和 Command 三种检测方式。

## HTTP 方式

```yaml
          livenessProbe:
            failureThreshold: 3
            httpGet:
              path: /index.html
              port: 80
              scheme: HTTP
            initialDelaySeconds: 20
            periodSeconds: 10
            successThreshold: 1
            timeoutSeconds: 1
          readinessProbe:
            failureThreshold: 3
            httpGet:
              path: /index.html
              port: 80
              scheme: HTTP
            initialDelaySeconds: 20
            periodSeconds: 10
            successThreshold: 1
            timeoutSeconds: 1
```

## TCP 方式

```yaml
          livenessProbe:
            failureThreshold: 3
            initialDelaySeconds: 60
            periodSeconds: 10
            successThreshold: 1
            tcpSocket:
              port: 80
            timeoutSeconds: 1
          ports:
            - containerPort: 80
              hostPort: 9654
              name: http
              protocol: TCP
          readinessProbe:
            failureThreshold: 3
            initialDelaySeconds: 20
            periodSeconds: 20
            successThreshold: 1
            tcpSocket:
              port: 80
            timeoutSeconds: 1
```

## Command 方式

```yaml
          livenessProbe:
            exec:
              command:
                - ls /usr
                - ls /mnt
            failureThreshold: 3
            initialDelaySeconds: 20
            periodSeconds: 10
            successThreshold: 1
            timeoutSeconds: 1
          readinessProbe:
            exec:
              command:
                - ls /mnt
                - ls /usr
            failureThreshold: 3
            initialDelaySeconds: 20
            periodSeconds: 10
            successThreshold: 1
            timeoutSeconds: 1
```

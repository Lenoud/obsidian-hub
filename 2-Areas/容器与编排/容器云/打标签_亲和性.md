# 节点标签与亲和性调度

使用 kubectl label 为节点打标签，通过 nodeSelector 和 nodeAffinity 实现 Pod 的定向调度。

## 节点标签操作

```bash
# 打标签
kubectl label node k8s-master-node1 disktype=ssd
kubectl label node k8s-master-node1 app=exam

# 查询标签
kubectl get nodes --show-labels | grep disktype=ssd
kubectl get nodes --selector disktype=ssd
kubectl get node k8s-master-node1 --show-labels | grep app=exam

# 取消标签
kubectl label node k8s-master-node1 app-
```

## nodeSelector + nodeAffinity 示例

同时指定 nodeSelector 和 nodeAffinity 时，两者必须都满足。

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: my-label
  labels:
    env: test
spec:
  containers:
  - name: my-ssd
    image: nginx
  nodeSelector:       # 部署到有此标签的节点
    disktype: ssd
  affinity:
    nodeAffinity:
      preferredDuringSchedulingIgnoredDuringExecution:  # 软条件（优先但不强制）
      - weight: 1
        preference:
          matchExpressions:
          - key: exam
            operator: In
            values:
            - chinaskill
      requiredDuringSchedulingIgnoredDuringExecution:   # 硬条件（必须满足）
        nodeSelectorTerms:
        - matchExpressions:
          - key: gpu
            operator: In
            values:
            - "true"
```

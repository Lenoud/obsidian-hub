# 污点（Taint）设置

使用 Kubernetes Taint 限制 Pod 调度到特定节点，配合 Toleration 实现灵活的调度策略。

## 污点操作

```bash
# 添加污点
kubectl taint nodes <node-hostname> node-role.kubernetes.io/master=:NoSchedule
kubectl taint nodes <node-hostname> toolServer=toolServer:PreferNoSchedule

# 删除污点
kubectl taint nodes <node-hostname> node-role.kubernetes.io/master-
```

## DaemonSet 容忍所有污点

确保 DaemonSet 在每个节点上运行，容忍所有污点：

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: nginx
spec:
  selector:
    matchLabels:
      name: nginx
  template:
    metadata:
      labels:
        name: nginx
    spec:
      containers:
      - image: nginx
        name: nginx
      tolerations:
      - operator: Exists
```

# KubeVirt 存储卷类型

KubeVirt 存储卷类型相关技术笔记。

虚拟机镜像（磁盘）是启动虚拟机必不可少的部分，KubeVirt 中提供多种方式的虚拟机磁盘，使用方式非常灵活。

## 说明

| 卷类型 | 数据持久性 | 说明 |
| :--- | :--- | :--- |
| cloudInitConfigDrive | - | 通过文件系统给 cloud-init 提供 meta-data 和 user-data |
| cloudInitNoCloud | - | 通过文件系统给 cloud-init 提供 meta-data 和 user-data，格式与 ConfigDrive 不同 |
| containerDisk | 不持久 | 指定包含 qcow2 或 raw 格式的 Docker 镜像，重启 VM 数据会丢失 |
| dataVolume | 持久 | 动态创建 PVC 并用指定磁盘映像填充，重启 VM 数据不会丢失 |
| emptyDisk | 不持久 | 从宿主机分配固定容量空间，重启 VM 数据会丢失 |
| ephemeral | 不持久 | VM 启动时创建临时卷，关闭后自动销毁 |
| hostDisk | 持久 | 在宿主机上创建 img 镜像文件给 VM 使用 |
| persistentVolumeClaim | 持久 | 指定 PVC 创建块设备 |
| secret | - | 使用 Kubernetes Secret 存储敏感数据并注入 VM |
| configMap | - | 把 ConfigMap 信息写入 ISO 磁盘挂给 VM |
| serviceAccount | - | 把 ServiceAccount 信息写入 ISO 磁盘挂给 VM |
| sysprep | - | 以 secret 或 configMap 形式往 VM 写入 sysprep |

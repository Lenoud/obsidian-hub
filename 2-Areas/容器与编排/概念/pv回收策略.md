# PV 回收策略

PV 回收策略相关技术笔记。

Kubernetes 中 `persistentVolumeReclaimPolicy` 决定了当一个持久化卷（PV）从它的声明（PVC）中释放时会发生什么。

## 说明

| 策略 | 行为 | 安全性 |
| :--- | :--- | :--- |
| Retain（默认） | PV 保留在 Released 阶段，需管理员手动处理 | 最安全 |
| Delete | PV 从 Kubernetes 中被删除 | 需谨慎使用 |
| Recycle | PV 重新回收到未绑定的持久化卷池中 | 可能存在数据泄露风险 |

## 详细说明

1. **Delete**：如果回收策略设置为 "Delete"，那么当 PV 从它的声明中释放时，它将会从 Kubernetes 中被删除。使用的卷插件必须支持删除操作才能生效。

2. **Recycle**：如果回收策略设置为 "Recycle"，那么当 PV 从它的声明中释放时，它将会重新回收到未绑定的持久化卷池中。使用的卷插件必须支持回收操作才能生效。

3. **Retain**（默认值）：如果回收策略设置为 "Retain"，那么当 PV 从它的声明中释放时，它将会保留在当前阶段（Released），需要管理员手动处理卷的回收工作。

> 从安全性的角度来看，"Retain" 策略通常被认为是最安全的选项，因为管理员需要手动处理卷的回收，确保数据的安全性和隐私。

# Kubernetes(K8S）所有核心服务服务的搭建

Kubernetes(K8S）所有核心服务服务的搭建相关技术笔记。

Kubernetes 容器编排相关笔记。

Kubernetes 是现代云原生架构中不可或缺的重要组件，而其核心服务（如 Service、StatefulSet、ConfigMap 等）通过 YAML 文件实现配置和管理。对于初学者而言，理解这些 YAML 文件的核心字段不仅能帮助快速上手 Kubernetes，也能为日后运维与扩展奠定坚实的基础。

在本文中，我们将系统地梳理 Kubernetes 核心服务的 YAML 文件格式，剖析其中的关键字段，帮助您从“知其然”到“知其所以然”，掌握 Kubernetes 核心配置的精髓。

### **Service**
**
**

**背景**：
在 Kubernetes 中，Pod 是短暂且可替换的，可能会因为部署、扩缩容或更新而动态改变 IP 地址。这使得直接与 Pod 通信变得不可靠。
**
**

**Service 的作用**：
Service 为一组动态变化的 Pod 提供稳定的网络访问接口，通过一个固定的虚拟 IP 和 DNS 名称，实现在集群内部或外部访问服务。

**典型场景**：

+ 将后端服务暴露给前端应用。
+ 为微服务架构中的服务间通信提供负载均衡。

用于将 Pod 暴露为网络服务。

### **StatefulSet**
**
**

**背景**：
某些应用（如数据库、分布式存储系统）对持久化数据存储、网络标识的稳定性有严格要求。无状态的 Deployment 无法满足这些需求。

**StatefulSet 的作用**：

+ 确保 Pod 的稳定网络标识（如 `pod-0`, `pod-1`）。
+ 提供数据持久化支持，Pod 重建后仍能挂载相同的存储卷。
+ 保障启动顺序和停止顺序。

**
**

**典型场景**：

+ 部署数据库（如 MySQL、PostgreSQL）。
+ 部署分布式存储系统（如 HDFS、Ceph）。

用于有状态应用的部署。

![[1735033625724-af7198ec-c034-4a81-bb66-eb9f5b78a3d1.png]]

### **数据持久化**
**背景**：
Pod 的生命周期是短暂的，当 Pod 删除或重建时，其内部存储的数据会丢失。而某些应用需要长期保存数据。
**
**

**PV 和 PVC 的作用**：

+ **PersistentVolume (PV)**：管理员创建的集群级别存储资源，定义存储特性（大小、访问模式等）。
+ **PersistentVolumeClaim (PVC)**：用户请求的存储资源，绑定到具体的 PV。

**典型场景**：

+ 持久化数据库存储。
+ 存储用户上传的文件、图片。

### **ConfigMap & Secret （应用配置）**
**
**

**背景：**

现代应用需要大量的配置项（如数据库连接信息、应用程序参数）。为了实现配置与代码解耦，Kubernetes 提供了 ConfigMap 和 Secret。

**ConfigMap**

**作用**：

ConfigMap 用于存储非敏感的配置信息，以键值对的形式组织，帮助应用程序与配置解耦。
**典型场景**：

+ 存储环境变量（如数据库连接地址、应用程序配置项）。
+ 配置文件注入（如 YAML 或 JSON 配置文件）。

![[1735033625886-bba6e406-734e-4cf9-9986-0ed1f2699b13.png]]

#### **Secret**
**
**

**作用**：

Secret 用于存储敏感信息（如密码、API 密钥、TLS 证书），以安全的方式注入到容器中。
**
**

**典型场景**：

+ 存储数据库密码、访问令牌。
+ 配置 HTTPS 所需的证书与私钥。

![[1735033626090-ec4af7e4-0aac-484b-a267-5cb2bc0eb00b.png]]

### **Namespace**
**
**

**背景**：

当一个 Kubernetes 集群中运行多个团队或项目时，资源管理可能变得复杂且容易冲突。
**
**

**Namespace 的作用**：

+ 提供逻辑分组，隔离不同团队或项目的资源。
+ 支持配额和访问控制。

**
**

**典型场景**：

+ 开发、测试、生产环境隔离。
+ 多团队共用一个集群时实现资源隔离。

![[1735033626207-1186a97c-150c-45cd-a1ab-38e5c95f58b7.png]]

### **Ingress**
**
**

**背景**：

传统的 Service 类型（如 NodePort, LoadBalancer）只能通过 IP 和端口访问应用，不适合复杂的 HTTP 路由场景。

**Ingress 的作用**：

+ 提供基于域名和路径的 HTTP/HTTPS 路由。
+ 支持负载均衡、TLS 终结等功能。

**
**

**典型场景**：

+ 使用单个域名路由多个服务。
+ 为 HTTPS 服务提供 TLS 加密支持。

![[1735033626162-cde808db-35be-455e-998a-924f927ffcc0.png]]

### **DaemonSet**
**
**

**背景**：

有些任务需要在每个节点上运行（如日志收集、监控代理），但这些任务不需要扩缩容功能。

**DaemonSet 的作用**：
确保集群中的每个节点运行一个 Pod，随着节点的新增或删除自动调整。

**典型场景**：

+ 日志采集工具（如 Fluentd、Filebeat）。
+ 系统监控工具（如 Prometheus Node Exporter）。

### 结尾

通过以上内容，我们系统地梳理了 Kubernetes 核心服务 YAML 文件的结构和关键字段。从 Service 到 StatefulSet，从 ConfigMap 到 Ingress，这些工具为我们提供了高度灵活且强大的资源管理能力。在实际使用中，熟练掌握这些字段可以帮助您在部署、扩展和优化应用时事半功倍。

Kubernetes 的魅力在于其强大的抽象能力，它将复杂的分布式系统管理转化为一个个清晰可配置的资源对象。希望本文的总结能为您深入理解 Kubernetes 核心服务提供帮助。

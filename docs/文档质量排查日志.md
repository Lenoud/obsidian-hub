# 文档质量排查日志

> 本文档是 Obsidian 仓库文档质量优化的**唯一事实来源**,记录所有排查过程与修复动作。
> 阅读顺序:[已完成总结](#已完成) → [目录扫描清单](#目录扫描清单) → [待修复队列](#待修复队列)

## 方法论

对每个目录的扫描覆盖以下 5 个维度,发现问题立刻修复或加入待修复队列:

1. **引用断链** — `[[xxx]]` 指向不存在的文件
2. **内容质量低** — 空壳、复制粘贴残留、片段化、命令堆砌
3. **时效性/正确性** — 命令、配置、版本号与官方文档不一致,或明显的错误
4. **格式问题** — 代码块标签错误、frontmatter 异常、标题层级混乱
5. **错别字与表达** — 语句不通、错别字、描述不清

修复原则:**每个逻辑组一个 commit**,commit 信息说明本组动作。

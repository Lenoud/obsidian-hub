# 系统还原

使用虚拟化平台将案件磁盘数据还原为可运行的虚拟机，支持 vmdk、qcow2、raw 等磁盘格式。

## Proxmox VE 虚拟化服务器

提供计算资源（CPU、内存等），支持 vmdk、qcow2、raw 磁盘格式的直接还原，无需额外转换。

如果需要还原到本地，可以在服务器还原成功后，克隆虚拟机成 vmdk 的磁盘，再拷贝出来，本地 VMware 还原也是可以的。

### 还原系统

回到 PVE 虚拟化服务器，点击创建虚拟机。

操作系统选项卡中，选择不使用任何介质。如果是 Windows 操作系统还原，在操作系统选项中类别修改成 Windows。

其它选项按照常规操作来即可，需要注意的是磁盘选项中，选择对应格式的磁盘和存储位置，选择网络磁盘。

![[1724292155456-ff86ff6f-4b39-43b4-a032-0fc72538008a.png]]

创建完成后虚拟机会有一个编号，在网络磁盘中 images 文件夹内会生成对应编号的目录，这个目录就是存放虚拟机磁盘的目录。因为网络协议对应的用户权限的原因，无法直接在 Windows 操作 PVE 的对应目录。

![[1724292620163-c1a77945-657f-4ed2-8c83-7041235b0424.png]]

接下来将下载完成的磁盘文件 copy 到对应的目录下，将下载下来的源磁盘名称修改成目录里面对应的磁盘名称，即实现了备份源数据的操作。

建议连接 192.168.1.201 服务器在终端操作，可以免去网络传输限制的速度，加快 copy 速度，降低网络占用。

root/qwer1234

![[1724293201826-a3281a9b-7e23-4d4b-b592-d8180692ccd7.png]]

![[1724293294518-1c7f364c-a7d9-45e6-88f1-5bf6f5282dbc.png]]

赋予磁盘全部权限：

```bash
chmod 777 ./100/vm-100-disk-0.qcow2
```

回到 PVE 正常开机还原即可。

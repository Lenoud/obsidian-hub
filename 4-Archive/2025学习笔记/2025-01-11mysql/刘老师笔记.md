4. 事务日志/重做日志(redo log)

	MySQL存储引擎Innodb，如果不小心崩溃了

	redolog让服务器拥有崩溃恢复能力

	什么时候写redo日志：

		事务开始之后，就产生redo log,

		在COMMIT之前 SQL产生的数据称为“脏数据” -> 这些数据没有写入到磁盘

		当脏数据写入到磁盘之后（COMMIT后），redo log就结束了

开启事务

BEGIN;

INSERT ...

INSERT ...

INSERT ...

DELETE ...

UPDATE ...

INSERT ...

UPDATE ...

提交事务

COMMIT;

up

redo log buffer中的内容什么时候写入磁盘

* 系统默认后台线程1s刷盘一次

innodb_flush_log_at_trx_commit：

0.设置为0的时候，表示每次事务提交时，不进行刷盘操作(1s)

1.设置为1的时候，表示每次事务提交时，进行一次刷盘（默认操作）<1s

2.设置为2的时候，表示每次事务时只把redo log buffer 写入 page cache (1s)

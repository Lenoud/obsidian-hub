# MySQL 事务

事务（Transaction）是一组被视为逻辑单元的数据库操作，要么全部执行成功，要么全部回滚。事务保证数据的一致性和完整性，特别是在并发环境下。

## ACID 四大特性

| 特性 | 说明 |
| --- | --- |
| **A**tomicity（原子性） | 事务中的操作要么全部成功，要么全部失败 |
| **C**onsistency（一致性） | 事务前后数据从一个一致状态变为另一个一致状态 |
| **I**solation（隔离性） | 并发事务相互独立运行，中间状态对其他事务不可见 |
| **D**urability（持久性） | 事务提交后，数据永久保存 |

## 并发问题

| 问题 | 说明 |
| --- | --- |
| 脏读（Dirty Read） | 读取到了其他事务未提交的数据 |
| 不可重复读（Non Repeatable Read） | 同一事务内前后多次读取，数据内容不一致 |
| 幻读（Phantom Read） | 同一事务内前后多次读取，数据总量不一致 |
| 丢失更新（Lost Update） | 多个事务同时更新同一数据，后提交的覆盖先提交的 |

## 四种隔离级别

| 级别 | 脏读 | 不可重复读 | 幻读 | 丢失更新 | 性能 |
| --- | --- | --- | --- | --- | --- |
| Read Uncommitted（RU） | 未解决 | 未解决 | 未解决 | 已解决 | 最好 |
| Read Committed（RC） | 已解决 | 未解决 | 未解决 | 已解决 | 较好 |
| **Repeatable Read（RR）** | 已解决 | 已解决 | 已解决 | 已解决 | 较差 |
| Serializable | 已解决 | 已解决 | 已解决 | 已解决 | 最差 |

> MySQL 默认隔离级别为 **RR（Repeatable Read）**。

## 操作命令

```sql
-- 查看当前隔离级别
SELECT @@tx_isolation;

-- 查看全局隔离级别
SELECT @@global.tx_isolation;
SHOW VARIABLES LIKE "%tx_is%";

-- 查看当前连接隔离级别
SELECT @@session.tx_isolation;

-- 设置隔离级别
SET GLOBAL TRANSACTION ISOLATION LEVEL READ COMMITTED;
SET SESSION TRANSACTION ISOLATION LEVEL READ UNCOMMITTED;  -- RU
SET SESSION TRANSACTION ISOLATION LEVEL READ COMMITTED;    -- RC
SET SESSION TRANSACTION ISOLATION LEVEL REPEATABLE READ;   -- RR
SET SESSION TRANSACTION ISOLATION LEVEL SERIALIZABLE;

-- 事务操作
BEGIN;                          -- 开启事务
UPDATE product SET price=price+100 WHERE pid=1;
SELECT * FROM product;
ROLLBACK;                       -- 回滚
COMMIT;                         -- 提交

-- 多回滚点
SAVEPOINT v1;
SAVEPOINT v2;
SAVEPOINT v3;
ROLLBACK TO v1;                 -- 回滚到指定保存点
```

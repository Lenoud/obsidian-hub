# MySQL 增量备份

使用 mysqldump + binlog 实现 MySQL 数据库的增量备份与恢复。

## 前提条件

需要开启二进制日志（binlog）：

```sql
-- 查看是否开启
SHOW VARIABLES LIKE 'log_bin';

-- 查看当前 binlog 文件
SHOW MASTER STATUS;
```

## 全量备份

```bash
# 全量备份所有数据库
mysqldump -uroot -p12345678 --all-databases --single-transaction --flush-logs --master-data=2 > full_backup_$(date +%Y%m%d).sql

# 全量备份单个库
mysqldump -uroot -p12345678 --single-transaction --flush-logs --master-data=2 testdb > full_testdb_$(date +%Y%m%d).sql
```

| 参数 | 说明 |
| --- | --- |
| `--single-transaction` | InnoDB 一致性快照，不锁表 |
| `--flush-logs` | 备份后切换新的 binlog 文件 |
| `--master-data=2` | 在备份文件中记录 binlog 位置（注释形式） |

## 增量恢复流程

```bash
# 1. 先恢复全量备份
mysql -uroot -p12345678 < full_backup_20250113.sql

# 2. 查看全量备份记录的 binlog 位置
head -50 full_backup_20250113.sql | grep "MASTER_LOG_POS"

# 3. 在 MySQL 中查看 binlog 文件列表
# mysql -uroot -p12345678 -e "SHOW BINARY LOGS;"

# 4. 从指定位置重放 binlog（增量部分）
mysqlbinlog --start-position=154 binlog.000003 | mysql -uroot -p12345678

# 5. 如需指定时间范围恢复
mysqlbinlog --start-datetime="2025-01-13 10:00:00" --stop-datetime="2025-01-13 15:00:00" binlog.000003 | mysql -uroot -p12345678
```

## 定时全量 + 增量方案

```bash
# 每天凌晨全量备份（crontab）
0 2 * * * mysqldump -uroot -p12345678 --all-databases --single-transaction --flush-logs --master-data=2 > /backup/full_$(date +\%Y\%m\%d).sql

# 定期清理旧 binlog（保留7天），在 MySQL 中执行：
# mysql -uroot -p12345678 -e "PURGE BINARY LOGS BEFORE DATE_SUB(NOW(), INTERVAL 7 DAY);"
```

## 说明

| 步骤 | 命令 |
| --- | --- |
| 全量备份 | `mysqldump --single-transaction --flush-logs` |
| 查看 binlog 位置 | `SHOW MASTER STATUS` |
| 重放 binlog | `mysqlbinlog --start-position=N binlog_file \| mysql` |
| 按时间恢复 | `mysqlbinlog --start-datetime --stop-datetime` |
| 清理旧日志 | `PURGE BINARY LOGS BEFORE` |

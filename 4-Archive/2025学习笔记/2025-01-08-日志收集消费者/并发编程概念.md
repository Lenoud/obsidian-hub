# Python 并发编程概念

Python 并发编程核心概念速查，涵盖多进程、多线程、协程三种模型。

## 进程 vs 线程 vs 协程

| 概念 | 说明 | 切换者 | 开销 |
| --- | --- | --- | --- |
| 进程（Process） | OS 分配资源的最小单位，独立内存空间 | 操作系统 | 高 |
| 线程（Thread） | CPU 调度的最小单位，共享进程内存 | 操作系统 | 中 |
| 协程（Coroutine） | 用户态轻量级线程，程序控制切换 | 程序自身 | 低 |

## Python 并发模型选择

| 模块 | 适用场景 | 特点 |
| --- | --- | --- |
| `multiprocessing` | CPU 密集型 | 多进程，绕过 GIL |
| `concurrent.futures.ProcessPoolExecutor` | CPU 密集型 | 进程池封装 |
| `threading` | IO 密集型 | 多线程，受 GIL 限制 |
| `concurrent.futures.ThreadPoolExecutor` | IO 密集型 | 线程池封装 |
| `asyncio` | IO 密集型 | 协程，单线程高并发 |

## 多进程示例

```python
from multiprocessing import Process, Pool
import time

def worker(n):
    return n * n

# 方式1：手动创建进程
p1 = Process(target=worker, args=(5,))
p1.start()
p1.join()

# 方式2：进程池
with Pool(4) as pool:
    results = pool.map(worker, range(10))
    print(results)
```

## 多线程示例

```python
from threading import Thread, Lock
import time

lock = Lock()
counter = 0

def increment():
    global counter
    for _ in range(100000):
        with lock:          # 加锁防止竞争
            counter += 1

t1 = Thread(target=increment)
t2 = Thread(target=increment)
t1.start(); t2.start()
t1.join(); t2.join()
print(counter)  # 200000
```

## 协程示例

```python
import asyncio

async def fetch(url):
    print(f"开始请求 {url}")
    await asyncio.sleep(1)  # 模拟IO
    print(f"完成请求 {url}")
    return f"result-{url}"

async def main():
    # 并发执行多个协程
    results = await asyncio.gather(
        fetch("url1"),
        fetch("url2"),
        fetch("url3"),
    )
    print(results)

asyncio.run(main())
```

## 说明

| GIL 影响 | 说明 |
| --- | --- |
| CPU 密集型 | GIL 导致多线程无法并行，必须用多进程 |
| IO 密集型 | IO 期间释放 GIL，多线程和协程均可 |
| asyncio | 推荐 IO 密集型场景，性能最好 |

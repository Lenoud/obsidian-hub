##### 1.元字符方括号 []
##注意元字符在方括号内不会触发"特殊功能"它只是匹配字符本身

[akm$]会匹配  'a','k','m','$'

[a-z]会匹配 所有小写字母

[abc]会匹配  a b c

##### 2.元字符脱字符^
^在[]里面的开头时，可以表示匹配方括号中未列出的所有其他字符

Eg.[^5]  匹配除了'5'的任何字符

^在[]的外面时，表示从字符串开头开始匹配

##### 3.元字符美元符$
在[]内时表示表示没有特殊含义

在[]外时表示匹配字符串结束

##### 4.元字符 反斜杠\
在[]中反斜杠后跟元字符，元字符的特殊功能不会被触发

反斜杠后跟字符表示特殊意义，如表示十进制数

| \w  | 匹配任何字母数字和下划线 |
| --- | --- |
| \W | 匹配任何非字母数字和下划线 |
| \d | 匹配任何十进制数字相当于[0-9] |
| \D | 匹配任何非十进制数字相当于[^0-9] |
| \s | 匹配任意空白字符相当于[\t \n \r \f \v] |
| \S | 匹配任意非空白字符相当于[^\t \n \r \f \v] |
| \b | 匹配单词的开始或者结束 |
| \B | 匹配单词的非开始或者结束 |

##### 5.元字符 圆点.
匹配除了换行符以外的任何一个字符

如果设置re.DOTALL标志，将匹配换行符在内的任何字符

Eg: res.search(re,text,flag=re.S)等价

##### 元字符    *   +   ？
*用于指定它前一个字符匹配零次或者多次。例如： ca*t 将匹配 ct（0 个字符 a）， cat（1 个字符 a）， caaat（3 个字符a），等等

+用于指定前一个字符匹配一次或者多次，例如 ca+t 会匹配 cat 和 caaat，但不会匹配 ct。

?用于指定前一个字符匹配零次或者一次，例如： '小?甲鱼'——可以匹配'小甲鱼'，也可以匹配'甲鱼’。

##### 元字符   {m, n}
m 和 n 都是十进制整数，上边讲到的几个元字符都可以使用它来表达，它的含义是前一个字符必须匹配 m 次到 n 次之间。例如：

a/{1, 3}b——会匹配 a/b， a//b 和 a///b。但不会匹配 ab（没有斜杠），也不会匹配 a////b（斜杠超过三个）；

{, n} 相当于 {0, n}；

{m, } 相当于 {m, +无穷}；

{n}，则是重复前一个字符 n 次。

你可能已经发现了，其实 *、 + 和 ? 都可以使用 {m, n} 来代替。

{0,} 跟 * 是一样的；

{1, } 跟 + 是一样的；

{0, 1}跟 ? 是一样的。

不过还是鼓励大家记住并使用 *、 + 和 ?，因为这些字符更短易阅读

##### 元字符    |
|  表示匹配左右任意一个表达式

 元字符  ()

()表示分组，即将括号里面的正则表达式看作一个整体，匹配的时候需要括号里面完全匹配才算匹配成功

import re

s0 = '1ab'

r0 = re.search(r'1(cb)|(ab)',s0)

print(r0.group())

 

s = ‘AAbb aabb’

r = re.search(r‘(Ab+) (a+b)’,s)

print(r.group()) #取所有匹配到的分组

print(r.group(0))#取所有匹配到的分组

print(r.group(1)) #取第一个分组匹配到的

print(r.group(2)) #取第二个分组匹配到的

 

 

\A 匹配字符串开始位置，忽略多行模式

\Z 匹配字符串结束位置，忽略多行模式

\b 匹配位于单词开始或结束位置的空字符串

\B 匹配不位于单词开始或结束位置的空字符串

 

 

其他元字符

| \<number> | 引用编号为number对应的分组匹配到的字符串 |
| --- | --- |
| (?P<name>) | 分组起一个别名 |
| (?P=name) | 引用别名为name的分组匹配到的字符串 |

如下示例可以看出分组结合|的妙用， r‘1cb|ab’ 只能匹配到 1cbb 和1cab，但是 r‘1(cb|ab)’则表示匹配 1cb 和 1ab,是把 ab 和 cb 当做一个整体

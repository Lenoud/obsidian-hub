# 网络模型(OSI 与 TCP/IP)

OSI 七层模型和 TCP/IP 四层模型是理解网络协议栈的基础,各层各司其职、向上提供抽象。

## OSI 七层 vs TCP/IP 四层

| OSI 七层 | TCP/IP 四层 | 协议示例 | 设备 |
| --- | --- | --- | --- |
| 7. 应用层 | 应用层 | HTTP/HTTPS、DNS、FTP、SMTP、SSH | 网关 |
| 6. 表示层 | 应用层 | SSL/TLS、JPEG、ASCII | 网关 |
| 5. 会话层 | 应用层 | NetBIOS、RPC | 网关 |
| 4. 传输层 | 传输层 | **TCP**、**UDP**、SCTP | 端口(进程) |
| 3. 网络层 | 网际层 | **IP**(v4/v6)、ICMP、OSPF、BGP | **路由器**、L3 交换机 |
| 2. 数据链路层 | 网络接口层 | Ethernet、PPP、ARP、VLAN | **交换机**、网卡、网桥 |
| 1. 物理层 | 网络接口层 | 电信号、光纤、无线电 | 集线器、中继器、网线 |

## 数据封装过程

```text
应用层      DATA
            ↓ 加 TCP/UDP 头
传输层      Segment(段) / Datagram(数据报)
            ↓ 加 IP 头
网络层      Packet(包)
            ↓ 加帧头帧尾(MAC 地址)
数据链路层  Frame(帧)
            ↓ 转为电信号/光信号
物理层      Bit(比特流)
```

## 各层职责要点

| 层 | 核心职责 | 关键概念 |
| --- | --- | --- |
| 物理层 | 比特流传输 | 电压、频段、RJ45、光纤 |
| 数据链路层 | 相邻节点通信 | MAC 地址、ARP、CSMA/CD、VLAN |
| 网络层 | 跨网段寻址 | IP 地址、路由、TTL、子网划分 |
| 传输层 | 端到端通信 | 端口号、TCP 三次握手、UDP 无连接 |
| 会话层 | 会话管理 | 建立/维持/断开(实际由应用层做) |
| 表示层 | 数据格式转换 | 加密、压缩、编码 |
| 应用层 | 用户接口 | HTTP、DNS、FTP |

## 常见面试题

| 问题 | 答案 |
| --- | --- |
| ARP 协议工作在哪层? | 数据链路层(2 层),通过 IP 查 MAC |
| ICMP 工作在哪层? | 网络层(3 层),ping/traceroute 依赖它 |
| TCP 和 UDP 的区别? | TCP 面向连接、可靠、有序;UDP 无连接、不可靠、快 |
| 路由器工作在哪层? | 网络层(3 层),转发 IP 包 |
| 交换机工作在哪层? | 数据链路层(2 层),转发以太网帧(MAC) |
| 三层交换机和路由器的区别? | 三层交换机硬件转发更快,适合大型局域网;路由器灵活、广域网适用 |

## 与 TCP 三次握手的关系

TCP 工作在**传输层**(OSI 第 4 层),它向上为应用层提供可靠的字节流服务,向下使用 IP 的"尽力而为"服务。握手过程是传输层自己的可靠性保证机制。

22. 题目：括号匹配

题目描述：给定一个只包括`'('`,`')'`,`'{'`,`'}'`,`'['`和`']'`的字符串 s，判断字符串 s 是否有效。

有效字符串需满足：

1. 左括号必须用相同类型的右括号闭合。

2. 左括号和右括号必须以正确的顺序匹配。

3. 每个右括号都有一个对应的左括号。

• 示例 1：

• 输入：`s = "()"`

• 输出：`true`

• 示例 2：

• 输入：`s = "()[]{}"`

• 输出：`true`

• 示例 3：

• 输入：`s="(]"

• 输出：`false`

• 示例 4：

• 输入：`s = "([)]"`

• 输出：`false`

s = input()

# 使用列表作为栈

stack = []

# 遍历字符串中的每个字符

for char in s:

    if char in '([{':  # 如果是左括号，推入栈中

        stack.append(char)

    elif char in ')]}':  # 如果是右括号

        if not stack or char == ')' and stack[-1] != '(' or \

           char == '}' and stack[-1] != '{' or \

           char == ']' and stack[-1] != '[':

            print("false")

            break

        stack.pop()  # 弹出栈顶元素

else:

    if not stack:  # 如果栈为空，则所有括号都正确匹配

        print("true")

    else:

        print("false")

1. 进程和程序的区别

进程是程序的动态执行实例，占用系统资源（如内存、CPU），具有独立地址空间；程序是静态的二进制文件。例如，运行中的nginx进程是程序/usr/sbin/nginx的实例。

2. 进程和线程

进程是资源分配的基本单位，线程是CPU调度的最小单位。线程共享进程内存，适合并发执行。如Python多线程处理日志I/O时，主线程接收请求，子线程处理文件写入。

3. 项目中使用进程/线程

在Kafka日志收集项目中，用Python多线程实现并发消费：主线程监听Kafka消息，子线程清洗数据并写入MySQL，提升吞吐量30%。

4. 死锁

死锁是多个进程因争夺资源而相互等待的状态，需满足互斥、持有并等待、不可抢占、循环等待四个条件。例如，两个事务同时请求对方持有的行锁。

5. 死锁案例

在MySQL集群中，曾因两个事务交叉更新订单和库存表导致死锁。通过设置innodb_lock_wait_timeout=5缩短等待时间，并优化索引减少锁竞争。

6. 互斥锁

互斥锁用于线程同步，确保同一时间只有一个线程访问共享资源。在Python中，使用threading.Lock()控制多个线程对数据库连接池的访问。

7. 七层网络协议

从应用层到物理层依次为：应用（HTTP）、表示（加密）、会话（连接管理）、传输（TCP/UDP）、网络（IP）、数据链路（MAC）、物理（光纤）。例如，浏览器通过HTTP（应用层）发送请求，经TCP（传输层）分段后由IP（网络层）路由。

8. 七层协议的实际应用

在Web集群中，Nginx（应用层）处理HTTP请求，通过TCP（传输层）与后端服务器通信，数据经IP（网络层）路由至目标主机，最终通过物理层网线传输。

9. TCP和UDP的区别

TCP面向连接、可靠、有拥塞控制（如三次握手），适合文件传输；UDP无连接、不可靠、高效，适合视频流。例如，HTTP用TCP保证数据完整，DNS查询用UDP快速响应。

10. TCP/UDP使用场景

TCP：HTTPS（银行支付）、SSH（远程管理）；UDP：DNS（域名解析）、实时视频会议（丢包不重传）。

11. 拥塞控制

TCP通过慢启动、拥塞避免等算法动态调整发送窗口，避免网络过载。在压测中，观察到当并发超过1500时，TCP自动降低传输速率以缓解拥塞。

12. 事务、索引、主键

事务是数据库中一组不可分割的操作序列事务是原子性操作,要吗全部执行，要吗全部不执行，有ACID特性；索引加速查询（如SELECT * FROM users WHERE age=20）；主键唯一标识记录，如用户表的id字段。

13. JOIN和GROUP BY

JOIN用于关联多表（如SELECT u.name, o.order_id FROM users u JOIN orders o ON u.id=o.user_id）；GROUP BY按字段分组统计（如统计各部门人数：SELECT dept, COUNT(*) FROM employees GROUP BY dept）。

20. 迭代器和生成器

迭代器

● 定义：实现了迭代器协议（__iter__() 和 __next__() 方法）的对象。__iter__() 返回迭代器自身，__next__() 返回序列下一个值，无元素时抛 StopIteration 异常。

● 作用：用于遍历集合元素，能逐个处理元素，而非一次性加载所有元素，节省内存。

生成器

● 定义：一种特殊迭代器，创建更简单，可通过生成器函数（含 yield 关键字）或生成器表达式（类似列表推导式，用圆括号）创建。

● 作用：延迟计算，仅在需要时生成值，节省内存和计算资源。

对比

● 共同点：都支持迭代，可逐个处理元素，节省内存。

● 不同点：生成器创建方式更简洁，迭代器需手动实现协议方法。

Kafka 的结构

Kafka 由 Producer（生产者）、Broker（服务代理节点）、Consumer（消费者）、Topic（主题）、Partition（分区）、Replica（副本）、Consumer Group（消费者组）。还有zookeeper

Kafka 为什么吞吐量那么快

顺序写入：消息追加到文件末尾，利用磁盘顺序读写性能。

零拷贝技术：减少数据拷贝次数，直接从 OS 缓存发送数据。

批量发送与压缩：减少网络请求次数，降低传输数据量。

分区并发：通过分区实现多个消费者并行处理。

Kafka 和 RabbitMQ 的差异

消息模型：Kafka 是分布式流处理平台，支持发布-订阅模型；RabbitMQ 是消息代理，支持 AMQP 协议。

吞吐量：Kafka 高吞吐量，适合大规模数据处理；RabbitMQ 在高吞吐量场景下可能有瓶颈。

持久性：Kafka 消息持久化到磁盘；RabbitMQ 默认存储在内存，可配置持久化。

消息顺序：Kafka 保证分区内的消息顺序；RabbitMQ 不支持消息顺序保证。

Kafka 如何保障消息的顺序性

Kafka 通过分区和偏移量来确保消息顺序。在同一个分区中，消息是有序的，消费者按照消息的偏移量顺序消费。

MySQL 执行一条 UPDATE 语句背后发生了什么

解析 SQL：MySQL 解析语句，生成执行计划。

锁定行：根据条件锁定需要更新的行。

修改数据：在内存中修改数据。

写入日志：将修改记录到事务日志（Redo Log）。

刷盘：将修改后的数据写入磁盘。

提交事务：完成事务提交。

MySQL 5.7 和 8.0 的区别

性能优化：MySQL 8.0 在查询优化器和索引性能上有显著提升。

新特性：MySQL 8.0 支持窗口函数、通用表表达式等。

配置和管理：MySQL 8.0 改进了配置文件和参数管理。

即时 DDL 和普通 DDL 的区别

即时 DDL：快速完成，通常通过元数据修改实现，对表结构影响小。

普通 DDL：可能涉及数据复制和表重建，耗时长。

MySQL 锁的类型

共享锁（S 锁）：允许多个事务读取同一资源。

排他锁（X 锁）：独占资源，防止其他事务访问。

意向锁：表示事务对资源的锁定意图。

什么是幻读

幻读是指事务读取到其他事务插入的行，导致前后读取结果不一致。

Linux 内核态和用户态的功能

内核态：管理硬件资源、调度进程、处理系统调用。

用户态：运行应用程序、调用内核服务。

容器用到 Linux 的底层技术

Cgroups：限制资源使用。

Namespace：隔离资源。

UnionFS：分层存储。

TCP 和 UDP 能否监听同一端口

不能。TCP 和 UDP 是不同协议，监听端口时需要明确指定协议。

浅拷贝和深拷贝的区别

浅拷贝：复制对象的引用，不复制对象本身。

深拷贝：复制对象及其内部引用的所有对象。

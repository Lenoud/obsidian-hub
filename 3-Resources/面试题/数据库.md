# 数据库面试题

数据库面试常考题汇总,涵盖数据类型、索引、事务、锁、设计范式等。

## MySQL 有哪些数据类型?

### 数值类型

| 类型 | 大小 | 范围(有符号) | 范围(无符号) | 说明 |
| --- | --- | --- | --- | --- |
| `TINYINT` | 1 字节 | -128 ~ 127 | 0 ~ 255 | 小整数(状态字段) |
| `SMALLINT` | 2 字节 | -32768 ~ 32767 | 0 ~ 65535 | — |
| `MEDIUMINT` | 3 字节 | -2²³ ~ 2²³-1 | 0 ~ 2²⁴-1 | — |
| `INT` | 4 字节 | -2³¹ ~ 2³¹-1 | 0 ~ 2³²-1 | 常用整数 |
| `BIGINT` | 8 字节 | -2⁶³ ~ 2⁶³-1 | 0 ~ 2⁶⁴-1 | 大整数(雪花 ID) |
| `FLOAT` | 4 字节 | 单精度浮点 | — | 不推荐金融用 |
| `DOUBLE` | 8 字节 | 双精度浮点 | — | 同上 |
| `DECIMAL(p,s)` | 可变 | 定点数 | — | **金融场景必用** |

> 整数推荐 `INT UNSIGNED` 或 `BIGINT UNSIGNED`,避免负数。金额用 `DECIMAL(18,2)`。

### 字符串类型

| 类型 | 长度 | 存储 | 说明 |
| --- | --- | --- | --- |
| `CHAR(n)` | 固定 n 字符 | n × 字符字节 | 定长,适合固定长度(MD5/UUID) |
| `VARCHAR(n)` | 变长 | 实际 + 1-2 字节 | 不定长字符串,常用 |
| `TEXT` | 64KB | — | 长文本(文章正文) |
| `MEDIUMTEXT` | 16MB | — | 很长的文本 |
| `LONGTEXT` | 4GB | — | 极长文本 |
| `BLOB` | 64KB | — | 二进制(图片) |

### 时间日期类型

| 类型 | 大小 | 范围 | 说明 |
| --- | --- | --- | --- |
| `DATE` | 3 | '1000-01-01' ~ '9999-12-31' | 仅日期 |
| `TIME` | 3 | '-838:59:59' ~ '838:59:59' | 仅时间 |
| `DATETIME` | 8 | '1000-01-01 00:00:00' ~ '9999-12-31 23:59:59' | 日期+时间(推荐) |
| `TIMESTAMP` | 4 | '1970-01-01' ~ '2038-01-19' | 时间戳,有 2038 问题 |

> MySQL 5.6+ 的 `DATETIME` 支持自动更新 `ON UPDATE CURRENT_TIMESTAMP`,已基本替代 `TIMESTAMP`。

### JSON / 特殊类型

- `JSON`(MySQL 5.7+):JSON 文档,支持索引和查询
- `ENUM('a','b','c')`:枚举,实际存 TINYINT(1-2 字节)
- `SET('a','b','c')`:集合,位图存储

## 索引(高频考点)

### 类型

| 索引 | 说明 |
| --- | --- |
| **B+Tree 主键索引**(聚簇) | InnoDB 主键默认,叶子节点存完整行 |
| **二级索引**(非聚簇) | 叶子节点存主键,需"回表" |
| **联合索引** | 多列组合,**最左前缀**原则 |
| **覆盖索引** | 查询字段全在索引里,不回表 |
| **唯一索引** | 唯一约束 + 索引 |
| **全文索引** | 全文搜索(InnoDB 5.6+)|

### 失效场景(避免)

```sql
-- 索引失效的常见写法
WHERE date(create_time) = '2024-01-01'   -- 左侧函数运算
WHERE name LIKE '%abc'                   -- 前缀模糊
WHERE id + 1 = 10                        -- 左侧运算
WHERE name = 123                         -- 隐式类型转换(name 是 VARCHAR)
WHERE a = 1 OR b = 2                     -- OR 两侧不全有索引
```

## 事务(ACID)

| 特性 | 含义 | 实现机制 |
| --- | --- | --- |
| **A** 原子性 | 全成功或全回滚 | undo log |
| **C** 一致性 | 数据约束不被破坏 | 业务 + AID |
| **I** 隔离性 | 并发互不干扰 | 锁 + MVCC |
| **D** 持久性 | 提交后不丢 | redo log |

### 4 种隔离级别

| 级别 | 脏读 | 不可重复读 | 幻读 |
| --- | --- | --- | --- |
| READ UNCOMMITTED | ✅ | ✅ | ✅ |
| READ COMMITTED(Oracle/PG 默认) | ❌ | ✅ | ✅ |
| **REPEATABLE READ**(MySQL 默认) | ❌ | ❌ | ❌(Next-Key Lock) |
| SERIALIZABLE | ❌ | ❌ | ❌ |

## 数据库设计三范式

| 范式 | 要求 | 反例 |
| --- | --- | --- |
| 1NF | 字段不可再分 | "电话"字段存了多个号码 |
| 2NF | 非主键字段完全依赖主键 | 订单表里存商品名(只依赖商品 ID) |
| 3NF | 非主键字段不传递依赖 | 订单表里存客户地址(依赖客户 ID) |

实际项目**适度反范式化**(加冗余字段换取查询性能)。

## 高频面试题速答

| 问题 | 速答 |
| --- | --- |
| `DELETE` 和 `TRUNCATE` 区别? | DELETE 走事务、可回滚、可带 WHERE;TRUNCATE 重置表,快但不可回滚 |
| `WHERE` 和 `HAVING` 区别? | WHERE 在分组前过滤行,HAVING 在分组后过滤组 |
| `UNION` 和 `UNION ALL` 区别? | UNION 去重(慢),UNION ALL 保留所有(快) |
| InnoDB 和 MyISAM 区别? | InnoDB 支持事务、行锁、外键;MyISAM 不支持,但查询快(MyISAM 已淘汰) |
| `COUNT(*)` vs `COUNT(1)` vs `COUNT(字段)` | 前两者等价,优化器会选最小索引;COUNT(字段) 不计 NULL |
| 什么是 MVCC? | 多版本并发控制,通过 undo log + read view 实现非锁定读,提升并发 |

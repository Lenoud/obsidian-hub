4、内存buffer/cache是什么？过多如何处理？内存实际空闲率怎么计算？
Buffer（缓冲区）：主要用于存储磁盘块设备的元数据和临时数据，作为内存与磁盘之间的缓冲层，减少磁盘I/O次数
Cache（缓存）：主要用于缓存文件内容，当程序读取文件时，内核会将文件内容缓存到内存中，下次读取时可直接从内存获取
实际空闲内存 = Free + Buffers + Cached + SReclaimable(来自/proc/meminfo)
实际内存空闲率 = (实际空闲内存) / 总内存 × 100%

## Python 部分
1. **Python解释器有哪些？ CPython与PyPy的区别是什么？**
    - **常见解释器**：
        * **CPython**：官方标准实现，使用C语言编写；
        * **PyPy**：采用JIT技术（即时编译器），在某些场景下运行更快；
        * 还有**Jython**（运行在Java虚拟机上）、**IronPython**（运行在.NET平台上）等。
    - **区别**：
        * **执行方式**：CPython直接解释执行字节码，而PyPy通过JIT编译可以将热点代码编译成本地机器码。
        * **性能与兼容性**：PyPy在循环密集型任务上通常更快，但某些C扩展可能兼容性较差；CPython则拥有最广泛的扩展库支持。
citepython-interpreter-diff
2. **Python2与Python3的区别（列举7个）**
    - **打印语句**：Python2中是`print`语句，Python3中必须使用函数形式`print()`。
    - **整数除法**：Python2中`/`做整数除法（如`3/2 == 1`），Python3中默认返回浮点数（`3/2 == 1.5`）；
    - **Unicode支持**：Python2中字符串默认为字节串，需要使用`u"..."`表示Unicode，Python3中字符串默认是Unicode；
    - **range/xrange**：Python2有`range`（返回列表）和`xrange`（生成器），Python3中只有`range`，且行为类似于xrange；
    - **异常语法**：Python2中`except Exception, e:`，Python3中改为`except Exception as e:`；
    - **迭代器方法**：如`dict.keys()`、`dict.values()`在Python2返回列表，在Python3返回迭代器；
    - **标准库调整**：部分模块的名称、结构在Python3中有所调整，例如`ConfigParser`变为`configparser`。
citepython2-vs-python3
3. **Python中list和set的区别**
    - **list**：有序、可变、允许重复元素，可以通过下标索引；
    - **set**：无序、可变（但元素必须是可哈希的）、不允许重复元素，查找速度快，常用于去重和集合运算。
citepython-list-set
4. **Python装饰器的作用与写法**
    - **作用**：用于在函数或类定义时动态地增强其功能，例如日志、权限检查、缓存等；
    - **常见写法**：使用`@装饰器名`语法，例如：

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        # 执行前操作
        result = func(*args, **kwargs)
        # 执行后操作
        return result
    return wrapper

@decorator
def some_function():
    pass
```

    - **两层与三层装饰器的区别**：
        * **两层装饰器**：即外层装饰器返回内层函数，包装原函数；
        * **三层装饰器**：通常用于带参数的装饰器，第一层接收参数，第二层接收被装饰对象，第三层完成包装。
    - **装饰器可以装饰**：函数、方法、类（通过修改类的行为），甚至属性。
citepython-decorator
5. **Python中list的底层实现、扩容机制与字典原理**
    - **list**：底层是动态数组（连续内存块），当容量不足时，会按照一定的比例（通常超过当前长度的1/4到1/2）分配新的更大内存，并复制原数据。
    - **字典**：基于哈希表实现，通过哈希函数将键映射到数组的索引；为减少冲突，字典采用开放寻址和再散列（rehash）策略。
citepython-list-dict-impl
6. **yield关键字、迭代器与生成器的使用场景**
    - **yield**：在函数中用来返回一个生成器，每次调用生成器的`__next__()`时，从上次挂起的位置继续执行；
    - **迭代器**：任何实现了`__iter__`和`__next__`方法的对象，可以用于for循环；
    - **生成器的使用场景**：适用于数据流、文件读取、无限序列生成以及内存敏感的场景，能动态生成数据而非一次性加载全部数据。
citepython-yield
7. **异常捕获的用途、语法结构及常见异常类**
    - **用途**：用于捕获并处理程序运行时的错误，保证程序不会因异常而中断；
    - **语法结构**：`try...except...else...finally`

```python
try:
    # 可能出错的代码
except SomeException as e:
    # 异常处理
else:
    # 无异常时执行
finally:
    # 无论异常与否均执行
```

    - **常见异常类**：`Exception`（基类）、`ValueError`、`TypeError`、`KeyError`、`IndexError`等。
citepython-exception
8. **变量名的解析顺序**
    - 遵循**LEGB规则**：Local（局部作用域） → Enclosing（嵌套函数作用域） → Global（模块全局作用域） → Built-in（内置作用域）。
citepython-namespace
9. **模块查找顺序，相对导入与绝对导入的区别**
    - **模块查找顺序**：
        * 当前模块所在目录
        * 环境变量`PYTHONPATH`指定的目录
        * 标准库目录
        * .pth文件指定的目录
    - **相对导入**：使用`.`或`..`表示当前或上级包，适用于包内部；
    - **绝对导入**：使用完整模块路径，从项目根目录开始，适用于全局引用。
citepython-import
10. **__name__与__file__变量的作用**
    - **name**：在模块被直接执行时为`"__main__"`，被导入时为模块名称，可用于模块自测；
    - **file**：存储模块的文件路径，方便定位模块所在位置。
citepython-name-file
11. **什么是魔术方法，常用的有哪些？**
    - **魔术方法**：以双下划线开头和结尾的方法，用于实现特殊行为（例如运算符重载、对象初始化等）；
    - **常用的**：`__init__`、`__str__`、`__repr__`、`__add__`、`__iter__`、`__getitem__`等。
citepython-magic-methods
12. **面向对象中Python3使用的继承算法**
    - Python3采用**C3线性化算法**（也称为“方法解析顺序（MRO）”），用于解决多继承时的继承顺序问题。
citepython-c3-linearization
13. **什么是GIL？有何作用？**
    - **GIL（全局解释器锁）**：确保同一时间只有一个线程执行Python字节码，保证内存管理的线程安全；
    - **作用**：简化解释器实现，但限制了多线程在CPU密集型任务上的并行性。
citepython-gil
14. **Python中实现并发编程可用的库及多线程应用场景**
    - **库**：`threading`（多线程）、`multiprocessing`（多进程）、`asyncio`（协程）、`concurrent.futures`等；
    - **多线程适用场景**：主要用于I/O密集型任务，如网络请求、文件操作，因为GIL对I/O等待不会构成瓶颈。
citepython-concurrency
15. **Python协程底层实现使用的库**
    - 协程主要基于生成器（`yield`）及`async/await`语法，标准库中实现的是**asyncio**模块，其底层依赖于事件循环与`selectors`模块。
citepython-asyncio
16. **是否打包过Python模块包，如何打包？**
    - 常见方法是使用`setuptools`或`distutils`编写`setup.py`，生成wheel或源码分发包，并通过`pip`进行安装；
    - 还可以借助工具如`twine`上传到PyPI。
citepython-packaging
17. **如何使用正则表达式，表示空行和以#开头的行**
    - **正则表达式模块**：`re`
    - **匹配空行**：例如：`r'^\s*$'`
    - **匹配以#开头的行**：例如：`r'^\s*#'`
citepython-regex
18. **内存管理与深浅拷贝的区别**
    - **内存管理**：Python采用**引用计数**和**垃圾回收**机制；
    - **浅拷贝**：只复制对象的引用（即第一层），内部嵌套对象仍共享；
    - **深拷贝**：递归地复制对象及其所有子对象，彼此独立。
citepython-memory-copy
19. **文件操作：如何删除、创建、写入、读取文件**
    - 使用内置函数`open()`操作文件；
        * **创建/写入**：`open('filename', 'w')`或`'a'`模式；
        * **读取**：`open('filename', 'r')`；
    - 删除文件：使用`os.remove('filename')`。
citepython-file-io
20. **assert关键字的作用**
    - 用于在开发和调试过程中检查表达式是否为真，如果为假则抛出`AssertionError`，常用于代码自检。
citepython-assert
21. **什么是死锁？如何避免死锁？**
    - **死锁**：多个线程或进程相互等待对方持有的资源，导致系统无法继续执行。
    - **避免方式**：
        * 保证加锁顺序一致；
        * 使用超时机制；
        * 尽量减少锁的粒度或使用无锁算法。
citeconcurrent-deadlock

## Go 部分
1. **变量的申明方式及“:=”的使用范围**
    - **变量声明方式**：
        * 使用`var`关键字，如：`var a int = 10`
        * 使用简短声明`:=`（仅限于函数内部，自动推导类型），例如：`a := 10`
        * 在包级作用域中只能用`var`声明。
citego-variable-declaration
2. **函数能否嵌套？如何返回另一个函数？**

citego-nested-function

    - Go支持在函数内部定义匿名函数（嵌套函数）；
    - 可直接将匿名函数作为返回值，例如：

```go
func outer() func(int) int {
    return func(x int) int {
        return x * 2
    }
}
```

3. **main函数与init函数的作用及执行顺序**
    - **init函数**：用于初始化工作，会在包被导入时自动执行，可以有多个，但执行顺序依赖于包的依赖关系；
    - **main函数**：程序入口，所有init函数执行完毕后才执行。
citego-init-main
4. **go mod的作用及其相对于GOPATH的优势**
    - **go mod**：用于模块化管理项目依赖，支持版本控制；
    - **优势**：解决了依赖混乱、版本冲突的问题，实现更稳定和可重复构建，无需依赖全局GOPATH环境。
citego-mod
5. **模块中公有变量（函数）的定义**
    - 以**大写字母开头**的标识符为公有，例如：`var ExportedVar = 123`、`func ExportedFunc() {}`。
citego-export
6. **Go是否支持默认参数和可变长参数**
    - **默认参数**：Go不支持函数参数默认值；
    - **可变长参数**：使用语法`func f(args ...int)`支持。
citego-parameter
7. **defer语句的执行顺序**
    - 多个`defer`语句会**后进先出**（LIFO）执行。
citego-defer
8. **如何判断一个变量是否为int类型**
    - 可使用**类型断言**或**类型选择**（type switch）来判断；也可使用`reflect.TypeOf()`进行检查。
citego-type-check
9. **如何判断两个切片是否相等**
    - Go内置并不支持直接比较切片内容；
    - 可以通过遍历逐个比较，或使用`reflect.DeepEqual(slice1, slice2)`（性能较低）进行比较。
citego-slice-equal
10. **结构体打印时%v与%+v的区别**
    - `%v`：打印结构体的值；
    - `%+v`：打印结构体时会显示字段名及对应的值。
citego-struct-print
11. **Go是否支持面向对象，如何实现多态？**
    - Go没有传统意义上的面向对象（没有类），但支持**结构体**与**方法**；
    - **多态**通过接口（interface）实现，不同类型只要实现相同接口的方法，就能相互替换。
citego-oop
12. **map与slice的底层实现与扩容机制**
    - **Slice**：底层是数组，通过结构体记录指针、长度与容量；扩容时通常按一定策略（例如2倍扩容）重新分配内存并复制元素。
    - **Map**：基于哈希表实现，使用若干桶存储键值对，当负载因子达到一定阈值时触发扩容和再散列。
citego-map-slice
13. **Go的垃圾回收管理，GC的阶段有哪些**
    - Go使用**并发垃圾回收**，主要分为**标记**和**清扫**两个阶段，并在这基础上进行优化以降低停顿时间。
citego-gc
14. **空struct{}的作用及Go中的内存对齐**
    - **空struct{}**：不占用内存，常用于表示集合中的“存在”状态；
    - **内存对齐**：Go编译器会自动进行内存对齐优化，以提高内存访问效率。
citego-struct
15. **Go中是否有异常类型**
    - Go没有传统的异常处理机制，使用的是**panic/recover**机制以及显式的错误返回值。
citego-error-handling
16. **指针的作用及rune类型**
    - **指针**：用于存储变量的内存地址，允许间接修改变量；
    - **rune**：是`int32`的别名，主要用于表示Unicode字符。
citego-pointer
17. **uint型变量1与2相减结果是多少**
    - 无符号整数运算遵循**模运算**规则，1-2会发生**下溢**，结果为对应位数所能表示的最大值（例如在32位系统中为4294967295）。
citego-uint-underflow
18. **reflect库的作用**
    - 用于在运行时检查变量的类型和值，实现**反射**机制，可以动态地操作对象、调用方法等。
citego-reflect
19. **new函数与make函数的作用及区别**
    - **new**：用于分配内存，并返回指向类型零值的指针；
    - **make**：仅用于内置类型（slice、map、channel）的初始化，返回相应类型的初始化值（非指针）。
citego-new-make
20. **Go的高并发模型与sysmon协程的作用**
    - **高并发模型**：基于**goroutine**和**channel**实现，支持大规模并发轻量级线程；
    - **sysmon协程**：是Go运行时内部的系统监控协程，负责监控系统状态、调度垃圾回收、处理抢占和其他运行时任务。

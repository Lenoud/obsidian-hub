### 【题目1】RabbitMQ集群
使用提供的OpenStack私有云平台，创建三个centos7.5系统的云主机，使用http://<IP>/RabbitMQ.tar.gz中的软件包安装RabbitMQ服务，安装完毕后，搭建RabbitMQ集群，并打开RabbitMQ服务的图形化监控页面插件。集群使用普通集群模式，其中第一台做磁盘节点，另外两台做内存节点。

一、基础环境安装

1.三个节点修改该主机名（记得做云主机的网络时，内外网一起做）

[root@localhost ~]# hostnamectl set-hostname rabbitmq1

[root@localhost ~]# hostnamectl set-hostname rabbitmq2

[root@localhost ~]# hostnamectl set-hostname rabbitmq3

2.三个节点配置/etc/hosts文件

# cat /etc/hosts

172.129.10.32   rabbitmq1

172.129.10.13   rabbitmq2

172.129.10.3    rabbitmq3

3.三个节点关闭防火墙和selinux

# systemctl stop firewalld && systemctl disable firewalld

# sed -i "s/=enforcing/=permissive/" /etc/selinux/config

# setenforce 0

#getenforce

Permissive

# tar -zxvf rabbitmq-repo.tar.gz -C /opt/

# cd /etc/yum.repos.d/

# cat local.repo

name=rabbitmq

baseurl=file:///opt/rabbitmq-repo

gpgcheck=0

enabled=1

5.三个节点安装rabbitmq-server，并启动rabbitmq-server

# yum install -y rabbitmq-server

#systemctl start rabbitmq-server && systemctl enable rabbitmq-server

6.三个节点启用rabbitmq图形化控制界面,并重启rabbitmq-server

# systemctl restart rabbitmq-server

7.使用界面查看

二、rabbitmq集群部署

1.三个节点全部关闭rabbitmq-server服务

[root@rabbitmq1 ~]# systemctl stop rabbitmq-server

[root@rabbitmq2 ~]# systemctl stop rabbitmq-server

[root@rabbitmq3 ~]# systemctl stop rabbitmq-server

2.将rabbitmq1 节点的.erlang.cookie 文件，并将该文件复制到 rabbitmq2 和 rabbitmq3 节

点的/var/lib/rabbitmq/目录下

[root@rabbitmq1 ~]# scp /var/lib/rabbitmq/.erlang.cookie 172.129.10.13:/var/lib/rabbitmq/

[root@rabbitmq1 ~]# scp /var/lib/rabbitmq/.erlang.cookie 172.129.10.3:/var/lib/rabbitmq/

[root@rabbitmq2 ~]# chown rabbitmq:rabbitmq /var/lib/rabbitmq/.erlang.cookie

[root@rabbitmq3 ~]# chown rabbitmq:rabbitmq /var/lib/rabbitmq/.erlang.cookie

[root@rabbitmq2 ~]# rabbitmqctl start_app

6.将rabbitmq3节点作为ram节点加入rabbitmq集群

[root@rabbitmq3 ~]# rabbitmqctl stop_app

[root@rabbitmq3 ~]# rabbitmqctl join_cluster --ram rabbit@rabbitmq1

[root@rabbitmq3 ~]# rabbitmqctl start_app

### 【题目2】修改glance存储后端
在提供的OpenStack私有云平台，创建一台云主机（镜像使用CentOS7.5，flavor使用带临时磁盘50G的），配置该主机为nfs的server端，将该云主机中的/mnt/test目录进行共享（目录不存在可自行创建）。然后配置controller节点为nfs的client端，要求将/mnt/test目录作为glance后端存储的挂载目录。配置完成后提交controller控制节点的用户名、密码和IP地址到答题框。

说明:

1.nfs-server的防火墙请自行关闭

2.nfs-server不用配置/etc/hosts

3.nfs-server请自行配置yum

一、nfs的安装

1.在nfs-server上安装nfs和rpcbind,并设为开机自启(yum请自行配置)

[root@nfs-server ~]# yum install -y nfs-utils rpcbind  安装nfs和rpc服务

[root@nfs-server ~]# systemctl start rpcbind && systemctl enable rpcbind      启动nfs服务前要先启动rpcbind服务

[root@nfs-server ~]# systemctl start nfs-server && systemctl enable nfs-server

2.在nfs-server节点上分区，创建共享目录，并将分区挂载到共享目录上(sdb就是那个50G的临时磁盘)

[root@nfs-server ~]# fdisk /dev/sdb   省略过程，但要给完全部大小

[root@nfs-server ~]# mkfs.xfs /dev/sdb1

[root@nfs-server ~]# mkdir /mnt/test  创建共享目录

[root@nfs-server ~]# mount /dev/sdb1 /mnt/test

3.在nfs-server节点上配置nfs

[root@nfs-server ~]# cat /etc/exports  编辑/etc/exports文件为如下内容

/mnt/test 172.129.10.0/24(rw)   (controller和compute第一张网卡的网段)

[root@nfs-server ~]#exportfs -arv  使配置生效

[root@nfs-server ~]# systemctl reload nfs  重新加载nfs服务

4.在controller上安装nfs

[root@controller ~]# yum install -y nfs-utils rpcbind

[root@controller ~]# systemctl start rpcbind && systemctl enable rpcbind

[root@controller ~]# systemctl start nfs-server && systemctl enable nfs-server

Glance修改后端存储的参考文献：[https://www.cnblogs.com/dexter-wang/articles/12235752.html](https://www.cnblogs.com/dexter-wang/articles/12235752.html)

镜像默认放在/var/lib/glance/images/文件夹下，可以在/etc/glance/glance-api.conf进行修改,挂载也是挂在/etc/glance/glance-api.conf里指定的文件夹，也就是/var/lib/glance/images

二、修改glance的后端存储

1.在nfs-server节点上设置/mnt/test目录的权限

[root@nfs-server ~]# chown -R 161:161 /mnt/test/

2.在controller节点上修改/var/lib/glance目录的权限

[root@controller ~]# chown -R glance:glance /var/lib/glance/

3.在controller节点上设置挂载路径（到这里已经满分）

[root@controller ~]# cat /etc/fstab

172.129.10.30:/mnt/test         /var/lib/glance/images          nfs     defaults        0 0  (nfs-server的ip)

[root@controller ~]# mount -a

4.创建镜像

[root@controller ~]# glance image-create --name cirros --disk-format qcow2 --container-format bare --progress < /root/cirros-0.3.4-x86_64-disk.img

5.在nfs-server节点上的/mnt/test/目录下看是否存在镜像数据

[root@nfs-server ~]# ls /mnt/test/

63074475-aad1-4e98-93f7-adefffea26f4  7f81b301-f804-4716-bf60-0ac88ec24b68

6.按照评分标准进行检查

[root@controller ~]# mount

172.129.10.11:/mnt/test on /var/lib/glance/images type nfs4 (rw,relatime,vers=4.1,rsize=524288,wsize=524288,namlen=255,hard,proto=tcp,port=0,timeo=600,retrans=2,sec=sys,clientaddr=172.129.10.10,local_lock=none,addr=172.129.10.11)

[root@controller ~]# ll -d /var/lib/glance/

### 【题目4】redis主从
使用提供的OpenStack私有云平台，申请两台CentOS7.5系统的云主机，使用提供的http源，在两个节点自行安装redis服务并启动，配置redis的访问需要密码，密码设置为123456。然后将这两个redis节点配置为redis的主从架构。配置完成后提交redis主节点的用户名、密码和IP地址到答题框。

有防火墙记得关防火墙,而且节点间不可以用scp传redis.conf文件，因为每个节点的redis.conf配置文件不相同

1.两个节点配置/etc/hosts/文件(两个节点的/etc/hosts/文件要一致)

[root@redis-master ~]# cat /etc/hosts

172.129.10.40   redis-master

172.129.10.50   redis-slave

[root@redis-master ~]# scp /etc/hosts 172.129.10.23:/etc/hosts

2.Redis-master和redis-slave节点安装redis：

[root@redis-master ~]# yum install -y redis

[root@redis-slave ~]# yum install -y redis

3.在redis-master节点的/etc/redis.conf文件添加如下内容:

[root@redis-master ~]# cat /etc/redis.conf

…

#bind 127.0.0.1  找到bind 127.0.0.1这行并注释掉

protected-mode no  关闭保护模式

daemonize yes   开启守护进程

requirepass "123456" 设置访问密码

masterauth "123456"   设置主库密码

appendonly yes   开启AOF持久化

4.在redis-slave节点的/etc/redis.conf文件添加如下内容：

[root@redis-slave ~]# cat /etc/redis.conf

#bind 127.0.0.1  可以不做

protected-mode no  关闭保护模式

daemonize yes   可以不做

requirepass "123456" 可以不做

slaveof 172.129.10.40 6379  redis1节点的ip地址和端口号（可以写主机名）

masterauth "123456"   设置主库密码

appendonly yes   可以不做

5.两个节点启动redis

[root@redis-master ~]# systemctl start redis && systemctl enable redis

[root@redis-slave ~]# systemctl start redis  && systemctl enable redis

6.先在主节点 redis-master 中登录 redis-cli 并输入密码，查看主从复制的信息，命令

采样率修改：agent.sample_n_per_3_secs配置说明：

在访问量较少时，链路全量收集不会对系统带来太大负担，能够完整的观测到系统的运行状况。但是在访问量较大时，全量的链路收集，对链路收集的客户端（agent探针）、服务端（SkyWalking OAP）、存储器（例如说Elastcsearch）都会带来较大的性能开销，甚至会影响应用的正常运行。在访问量级较大的情况下，往往会选择抽样采样，只收集部分链路信息。SkyWalking Agent在agent/config/agent.config 配置文件中，定义了agent.sample_n_per_3_secs配置项，设置每3秒可收集的链路数据的数量。

### 【23国赛样题5】Minio服务搭建
使用提供的OpenStack云平台，申请一台云主机，使用提供的软件包安装部署MINIO服务并使用systemctl管理。

官方文档：[https://docs.minio.io/](https://docs.minio.io/)

Minio是一个开源的对象存储服务器软件

[root@minio ~]# tar -zxvf minio.tar.gz -C /opt/    #这里解压到什么位置自定义

[root@minio ~]# chmod +x /opt/minio    #赋予可执行权限

准备Minio.service

配置用户为admin：Environment=“MINIO_ROOT_USER=admin”

配置用户密码为12345678：Environment=“MINIO_ROOT_PASSWORD=12345678”

配置启动minio文件的路径以及存储数据的目录：ExecStart=/opt/minio server /data/minio

配置启动失败重启：Restart=on-failure RestartSec=15

[root@minio ~]# mkdir -p /data/minio  #创建数据存储路径，可以自定义路径

[root@minio ~]# cat /etc/systemd/system/minio.service

[Unit]

Description=MinioService

[Service]

Environment="MINIO_ROOT_USER=admin"

Environment="MINIO_ROOT_PASSWORD=12345678"   #不设置用户密码，默认用户密码是minioadmin

ExecStart=/opt/minio server /data/minio/

Restart=on-failure

[Install]

WantedBy=multi-user.target

启动Minio服务

启动服务：systemctl start minio.service

查看启动日志：journalctl -u minio.service -f

查看mino服务状态：systemctl status minio.service
